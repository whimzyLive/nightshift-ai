# Prettier fixed-point gate for plugin docs — Technical Spec

**Story:** NA-62 — Prettier 3.6.2 non-idempotency corrupts indented fenced command blocks in plugin docs — CI affected-only gate misses it
**Feature:** — (systemic-fix story, no parent Epic)
**Date:** 2026-07-18

## Overview

Prettier 3.6.2 silently **corrupts** a common markdown pattern in the plugin docs — it is **non-idempotent** on certain fenced code blocks, so a well-formed committed file is not a Prettier fixed point and the next `prettier --write` (a pre-commit hook, an editor-on-save, or any PR that happens to touch the file) shatters runnable command blocks into broken four-backtick fences with unterminated shell quotes. CI misses it because the format gate (`ci.yml` line 34) is **affected-only** (`--base`-scoped), so a latent file not in the current diff is never checked. This story reformats every currently-flagged block to a genuine Prettier fixed point and adds an **all-files `prettier --check`** over `plugins/**/*.md` so the class stops recurring.

## Scope of this story (authoring surface)

This is a **plugin-authoring + CI-infrastructure** story spanning **two owners**. It adds no product code, no database entity, no HTTP surface. It ships (1) a reformat sweep of the flagged files under `plugins/**`, (2) a thin wrapper script that runs the all-files check, and (3) the `.github/workflows/ci.yml` wiring that invokes it — the last being CI infrastructure, not a `plugins/` artifact.

### The three file states (the reasoning the gate and sweep are built on)

Every `plugins/**/*.md` file sits in one of three states w.r.t. the Prettier fixed-point predicate `format(X) == X`:

- **State 0 — clean fixed point.** `format(X) == X` and command blocks are whole. `prettier --check` passes. Nothing to do.
- **State 1 — well-formed but UNSTABLE (the landmine).** Command blocks are whole and runnable, but `format(X) == X'` where `X'` is the corrupt quad-fence form (`X' != X`). `prettier --check` **fails** ("not formatted"). The next unguarded `prettier --write` turns `X` into `X'`, shattering the block. This is the state that bit NA-56. **The all-files `--check` gate catches state 1.**
- **State 2 — already-corrupted but STABLE.** The shattered form `X'` is itself a fixed point (`format(X') == X'`), so `prettier --check` **passes** on it even though the command block is already broken. **Neither `--check` nor any write-on-copy pass flags state 2** — both only test the fixed-point predicate, which `X'` satisfies. Only a corruption-signature grep catches it (see the sweep).

Verified against the current tree: `prettier --check "plugins/**/*.md"` exits 1 and flags **44** state-1 files (invisible to the affected-only line-34 gate); the state-2 signature grep finds only legitimate nested-example fences (zero real corruption today).

## The four crux decisions

### 1. Detection method — all-files `prettier --check` (native), not a bespoke write-on-copy detector

**The gate is `prettier --check "plugins/**/*.md"`.** `prettier --check FILE` reports exactly `format(FILE) != FILE` natively — it is the fixed-point predicate, computed in one call, with no scratch copy, no `--file-info` guard, and no cleanup trap.

**Correcting an earlier framing error (mine, in the dispatch and the first draft of this spec):** "`--check` is insufficient because it passes on the corrupt form" conflated two states. `--check` "passes on the corrupt form" refers only to **state 2** (already-corrupted-but-stable) — and a write-on-copy pass **also** passes on state 2 (`X'` is a fixed point), so write-on-copy is no better there. The file that bit NA-56 was **state 1** (well-formed-but-unstable), and `prettier --check` **fails** on exactly those. The real CI hole was never `--check`; it was the **affected-only `--base="remotes/origin/main"` scoping** (line 34) skipping files not in the diff. So the fix is scope, not mechanism: run the same native `--check`, but over **all** `plugins/**/*.md` on every PR.

**Why not the write-on-copy pipeline the first draft proposed:** it re-implements `prettier --check` (this spec's own earlier text conceded the identical predicate). Its two justifications collapse under scrutiny:

- "Don't trust the exit code / stdout (RTK wrapper)" — RTK is the **local developer shell hook**; it is **absent on the GitHub Actions runner** where this gate runs. In CI, `prettier --check`'s exit code is authoritative.
- "It prints the diff" — developer ergonomics for the sweep, not gate correctness. That diagnostic value belongs to the **sweep** (Decision 2), not the CI gate.

Removing the write-on-copy pipeline also removes its whole failure surface: no scratch file, no NA-51 `--file-info`-`ignored:false` guard, no collision/cleanup trap. Simpler and more durable.

**Verification (state 1 caught):** run today, `prettier --check "plugins/**/*.md"` exits 1 and names 44 files — each a state-1 file `format(X) != X` that the affected-only line-34 gate never checks unless a PR happens to touch it. The redesigned gate flags all 44 on every PR. Reasoned through: a state-1 file `X` has `format(X) == X' != X`, so `--check` computes `X'`, compares `X' != X`, and reports "not formatted" → non-zero exit → gate fails. Confirmed empirically, not just argued.

### 2. The reformat sweep — the check is the finder; plus a state-2 signature grep

There is no separate audit step for state 1: **run `prettier --check "plugins/**/*.md"`; every file it names is a file to bring to a fixed point.** Triage each flagged file — you cannot blind-`prettier --write` the whole set, because a blind write is exactly what shatters the command-block files:

- Inspect Prettier's proposed change for the file (e.g. `prettier --write` on a throwaway copy, or a `git`-restored scratch, and read the diff — this is where write-on-copy legitimately belongs, as a **local one-time triage tool**, not the CI gate).
- **Benign reformat** (whitespace, list markers, table padding): accept the `prettier --write` output.
- **Command-block shatter** (the diff introduces four-backtick fences / splits a quoted command): do **not** apply the blind write. Manually reformat to a stable fixed point that preserves the runnable command — **dedent the fence to column 0** (and, per the NA-57 caveat, dedent any **trailing list-continuation prose after the fence** too, since a column-0 fence terminates the list item), **insert a blank line** between a paragraph and a following list (NA-61 variant), or **remove the embedded blank line** inside the quoted block — whichever keeps the command whole.
- Re-run `prettier --check` until it reports clean over `plugins/**/*.md`.

**The state-2 gap (independent of the gate) — MUST be in the sweep.** The fixed-point predicate does **not** flag a state-2 file (already-corrupted-but-stable — `X'` is a fixed point, so `--check` passes). If any such file is already in the tree, neither the gate nor any write-on-copy pass catches it. So the sweep must **also** grep the corruption signature across `plugins/**/*.md`, independent of the gate:

- Signature: **four-backtick fences** (` ```` `) and split / unterminated command blocks in command/bash contexts.
- Each hit is a **human-triage candidate, not an auto-fail** — four-backtick fences are legitimate when wrapping a nested markdown example that itself contains three-backtick fences. Verified: the only current hits (`plugins/sdlc/skills/writing-docs/SKILL.md`, `plugins/sdlc/skills/writing-specs/SKILL.md`, 4 occurrences) are exactly that legitimate usage — **zero real state-2 corruption in the tree today**. The grep is a manual audit the impl runs and inspects; it stays out of the CI gate precisely because it cannot distinguish legitimate nested fences from corruption automatically.

### 3. The CI gate change — supplement line 34 with an all-files check; keep it a thin wrapper script

**The hole:** line 34 runs `pnpm nx format:check --base="remotes/origin/main"` — affected-only. A latent file not in the current diff is never checked, so it ships green.

**The change:** add a new step invoking the guard, mirroring line 36:

```yaml
- run: pnpm nx format:check --base="remotes/origin/main"   # unchanged — supplement, not replace
- run: pnpm nx run-many -t lint test build typecheck e2e
- run: bash plugins/sdlc/scripts/check-agent-skill-preloads.sh
- run: bash plugins/sdlc/scripts/check-plugin-docs-format.sh   # NEW — all plugins/**/*.md, every PR
```

**Why plain `prettier`, not `nx format:check`, despite CLAUDE.md's nx preference.** `nx format:check` is **project-graph-based** — its scoping flags are `--all` ("Format all projects"), `--projects`, `--libs-and-apps`, and the affected `--base`/`--files`/`--uncommitted` set. `plugins/**/*.md` belong to **no nx project**, so there is no nx form that checks *every plugin markdown file by path*. `--all` formats project files (missing plugins md); `--base` is the affected-only hole itself. The correct tool for an all-files check over a path glob of non-project files is `prettier --check` directly. This is a deliberate, reasoned deviation from the nx-first guideline, not an oversight.

**Thin wrapper script vs. a raw `ci.yml` line — decision: keep the wrapper.** The gate body is now one `prettier --check` call, so a raw `- run: pnpm exec prettier --check "plugins/**/*.md"` line in `ci.yml` (platform-engineer only, no `plugins/` file) is a defensible smaller alternative. I keep a thin wrapper script because it (a) **single-sources** the `plugins/**/*.md` scope so the sweep-finder and the gate cannot drift, (b) mirrors the **established precedent** exactly (`check-agent-skill-preloads.sh`: guard logic in a `plugins/` script, invocation in `ci.yml`), keeping the two-agent ownership split clean and reviewable, (c) is the **locally-runnable finder** maintainers use during the sweep and future edits, and (d) carries the fail-fast empty-glob guard. The script is a **thin wrapper around the all-files check — not** the per-file write-on-copy pipeline the LEAD removed.

**Supplement, not replace (unchanged).** A fixed-point check and a `--check` are the same predicate; the only axes are file-set scope and affected-only vs. all-files. Over `plugins/**/*.md` the new gate subsumes line 34; over the rest of the repo (`.ts`, `.json`, non-plugin `.md`) line 34 is the only coverage, so it stays. Making line 34 all-files repo-wide is out of scope — it would trip on out-of-scope non-plugin latent files and block this story.

**`plugin.json` bump — resolved: no bump.** The sibling guard `check-agent-skill-preloads.sh` shipped in `c85ec50` with no `plugin.json` bump (verified: the intro commit's stat touches only the script). A CI guard is repo-infrastructure invoked by `ci.yml`, not consumer-facing plugin behaviour a pinned consumer pulls; and the sweep's reformats are cosmetic (they ship no new instructions — the semantic content is byte-preserved). Both the artifact table and this decision agree: **no `plugin.json` change.** (Earlier draft's Open Question on this is closed.)

### 4. Config vs. reformat — reformat + gate; do NOT rely on a Prettier config change

Investigated whether a config setting makes these blocks idempotent, which would be a smaller fix than reformatting every flagged block:

| Option                              | Effect on this corruption class                                                                                                                                                                                          | Verdict                           |
| ----------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------- |
| `proseWrap` (`preserve`/`never`/…)  | Governs prose line-wrapping, not fenced-code re-fencing. The corruption is remark mis-parsing an indented fence with an embedded blank line under a list item. No effect expected.                                       | Does not fix                      |
| `embeddedLanguageFormatting: "off"` | Disables formatting of code **inside** embedded blocks. The shatter is a **markdown-structure** (backtick-count / list-boundary) rewrite by the outer parser, not embedded-language reformatting. Unlikely to help.     | Speculative; not relied upon      |
| Pin/downgrade Prettier version      | Already `~3.6.2`; the bug is in this version. Non-idempotency bugs span versions — no version is provably idempotent, and a version change fixes none of the already-flagged files.                                       | Speculative; does not fix backlog |

**Recommendation: reformat the flagged blocks + the all-files `prettier --check` gate. Do not adopt a config change as the fix.** The reformat makes each block a fixed point that is stable **regardless of config**, and the gate is **root-cause-agnostic** — it catches this variant, the NA-61 variant, and any future non-idempotency, because it enforces the observable invariant (committed `== format(committed)`) rather than betting Prettier is globally idempotent. A config change would fix none of the 44 latent files and prove nothing about other inputs. Impl **may** empirically test `embeddedLanguageFormatting: "off"` as defense-in-depth (adopt only if it demonstrably makes the known blocks idempotent and churns no other file), but never as a substitute for the gate — see Open Question 2.

## `command-reference` auto-regen check (Standing lesson 5)

A new file under `plugins/sdlc/scripts/` was checked against the `command-reference` auto-regeneration surface. Per `plugins/sdlc/refs/doc-types.md`, the `command-reference` doc-type is generated from **command frontmatter + body** (`plugins/*/commands/*.md`) into `docs/reference/commands/`. A **script** under `plugins/sdlc/scripts/` is not a command and is not enumerated by that generator, so adding `check-plugin-docs-format.sh` triggers **no** `command-reference` regeneration. (The earlier draft's "confirmed by grep, only `ci.yml` references the sibling script" claim was false — `grep -rn check-agent-skill-preloads` returns ~30 hits across memories and plans — and is dropped; the conclusion rests on the generator's input set, verified against `doc-types.md`, not on a reference count.)

## Permissions Detail

Not a multi-role RBAC surface. The actors are the **CI runner** (executes the gate on every PR to `develop`) and the **founder** (reviews the two PRs). The one authority statement: the gate is **blocking** — a non-zero exit fails the `main` CI job, so a non-fixed-point `plugins/**/*.md` file cannot merge.

## Error Handling

`check-plugin-docs-format.sh` failure modes — every failure is a **loud, non-zero exit** (never a warning, never a silent no-op):

| Scenario                                                                       | Behaviour                                                                                                                                            | Exit |
| ------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------- | ---- |
| One or more `plugins/**/*.md` files are not Prettier fixed points               | `prettier --check` prints its native `[warn] <file>` list and exits non-zero; the wrapper surfaces that verbatim and fails.                          | 1    |
| The `plugins/**/*.md` glob matches no files (misconfigured path / wrong dir)    | **Hard FAIL** — mirror `check-agent-skill-preloads.sh`'s empty-set guard; a vacuously-green gate is worse than none. (Prettier's own no-match error also exits non-zero; the wrapper makes the intent explicit.) | 1    |
| Prettier / `node_modules` unavailable (`ERR_PNPM_RECURSIVE_EXEC_FIRST_FAIL`)    | **Hard FAIL** with an actionable message (run `pnpm install --frozen-lockfile`). CI installs at line 31, so this is a local-run guard.               | 1    |
| Every `plugins/**/*.md` file is a fixed point                                   | Print "OK"                                                                                                                                          | 0    |

## Out of Scope

- **Fixing corruption in non-plugin markdown.** `docs/superpowers/` and `docs/features/` are prettier-ignored. `apps/marketing/`, `tools/`, `brand/`, `packages/ui/` markdown belongs to **other agents' domains** — not touched here.
- **Closing the affected-only hole for non-plugin file types repo-wide.** Line 34 stays affected-only; widening it repo-wide risks tripping on out-of-scope non-plugin latent files and would block this story. A per-domain follow-up can address it.
- **Non-markdown fixed-point checking.** The gate is scoped to `plugins/**/*.md`; `.ts` / `.json` stay under line 34.
- **Changing or pinning a different Prettier version**, and **adopting a Prettier config change as the fix** — investigated and not adopted (Decision 4); config change permitted only as optional, empirically-verified defense-in-depth.
- **A pre-commit / local git hook.** The gate lives in CI (the enforcement point the story names).
