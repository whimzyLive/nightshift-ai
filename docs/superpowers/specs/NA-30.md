## Overview

Build the foundational layout shell for the four-page nightshift marketing site: the Tailwind v4 CSS-first `@theme` token layer plus the site-wide brand chrome (night-sky background, glass nav, glass footer, cursor glow, neon CTA button, hover-lift card) as reusable components wrapping every page. This is chrome + tokens only â€” no page-specific content, and no CMS.

## Scope & boundaries

- **This story:** the `(frontend)` layout shell, the `@theme` token config, and the reusable chrome/primitive components every later page story consumes.
- **Not this story:** any page body content (hero, FAQ entries, team roster, argument sections), CMS collections/globals, SEO/JSON-LD wiring, and the home-only heavy sky variant (crescent moon, meteor showers, mouse-parallax star layers). Those land in the sibling stories this one blocks (NA-32â€¦NA-38).
- **Layer:** web only. Owner agent: **web-engineer** (`apps/marketing/`, `packages/ui/`, `brand/`).

## Prerequisites (stack setup this story introduces)

`apps/marketing` is a Next.js 16 + React 19 + Payload 3 app with **no Tailwind installed yet** (`apps/marketing/src/app/global.css` is a bare reset; `packages/ui/src/index.ts` is an empty shell; the `packages/ui/dist/*` artifacts are stale from the superseded NA-20 3D-hero build and are **not** a source of truth). This story therefore also stands up Tailwind v4:

- Add Tailwind v4 + the PostCSS plugin as dev deps to `apps/marketing` (`tailwindcss`, `@tailwindcss/postcss`, `postcss`).
- Add a PostCSS config to `apps/marketing` that registers `@tailwindcss/postcss`.
- Tailwind v4 is **CSS-first**: no `tailwind.config.js`. All configuration (tokens, content sources, custom utilities) lives in CSS via `@import "tailwindcss"`, `@theme`, and `@source`.
- Register `packages/ui/src` as an additional content source (`@source` pointing at the ui package `src`) so utilities used inside shared components are generated.

Do **not** touch `apps/marketing/src/app/(payload)/custom.scss` â€” it is Payload's generated admin override (the SCSS-file exception noted in the root `CLAUDE.md`). No new `*.module.css` or `*.scss` files anywhere else.

### Data hooks / state

No server data â€” the chrome is fully static (Epic: "No server data â€” everything is static content + client state"). Client state is component-local only:

- `NavBar`: boolean "detached" state derived from a passive `scroll` listener (threshold ~320px).
- `CursorGlow`: pointer x/y + "over-interactive" size state from a passive `pointermove` listener; unmounted on coarse pointers / reduced motion.
- `NightSky`: animation driven by CSS keyframes (no JS state) except the reserved home variant.

No global store, no React Context needed for this story.

## Accessibility & reduced motion

`prefers-reduced-motion: reduce` must **statically disable** all motion while keeping content fully legible and reachable (AC ti-6):

| Element                        | Motion (default)            | Under `prefers-reduced-motion`                    |
| ------------------------------ | --------------------------- | ------------------------------------------------- |
| NightSky stars                 | twinkle/drift keyframes     | static starfield, no animation                    |
| CursorGlow                     | pointer-tracking glow       | not rendered at all                               |
| Meteors / home parallax / moon | (home story)                | static â€” flagged for the home story               |
| NavBar detach                  | transition on detach        | instant state change, no transition               |
| Card hover-lift                | translate + glow transition | color/border change only, no transform/transition |
| CtaButton                      | shadow/color transition     | end-state styling, no transition                  |

Prefer a single global `@media (prefers-reduced-motion: reduce)` guard in the base layer plus per-component reduced-motion checks where JS drives the effect. Focus-visible rings (`--glow-focus`) remain in all cases. Color pairings already meet WCAG AA per the token files â€” do not alter token values.

## Error Handling

Static presentational chrome; no network or validation surface. Edge cases:

| Scenario                                   | Behaviour                                                                                                                                     |
| ------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| JS disabled                                | Nav renders in its non-detached glass state; content and links fully usable; sky renders via CSS; cursor glow simply absent. No layout break. |
| Coarse pointer (touch)                     | `CursorGlow` renders nothing; no listeners attached.                                                                                          |
| `prefers-reduced-motion: reduce`           | All motion disabled per the table above; content unaffected.                                                                                  |
| Nav link to a not-yet-built anchor/section | Link resolves to the route; missing in-page anchor is a no-op scroll (acceptable until the section lands).                                    |
| Fonts fail to load                         | `display: swap` falls back to the system stack declared in `--font-sans`/`--font-mono`.                                                       |

No offline behaviour (no offline sync in this project; no mutations).
