# Creative motion enhancements for the marketing site — Technical Spec

**Story:** NA-69 — Add creative motion enhancements to marketing site
**Epic:** NA-29 — Nightshift marketing site — design-handoff build
**Feature:** docs/features/2026-07-12-nightshift-marketing-site-design-handoff.md
**Date:** 2026-07-23

## Overview

Layer eleven restrained, technical-feeling motion beats onto the already-shipped `apps/marketing`
site so the night-to-dawn "ships while you sleep" narrative is felt, not just read. Every beat builds
on the motion.dev v12 primitives, `EASE_OUT`/`--dur-*` tokens, and per-component reduced-motion latch
already merged under NA-29 — no new dependency, no new page, no data or API change.

## Scope classification

This is a pure UI/animation story. The following spec sections are **N/A**:

- **Data Model** — N/A. No entity, collection, field, or Payload schema change; all animated content
  is existing static/CMS-served copy.
- **API Surface** — N/A. No new route handler, endpoint, or request/response shape; motion runs
  entirely client-side.
- **Permissions** — N/A. Marketing site is public and unauthenticated; no role gates exist to change.
- **Offline Sync** — N/A. The project has no offline-sync layer (see project-context — web-only Next.js
  app).

The design surface is **Web UI + motion architecture**, specified per sub-item below.

## Shared conventions (apply to every sub-item)

These are the load-bearing invariants pulled from the existing kit. Every sub-item MUST honour them.

| Convention | Concrete value / source | Rule |
| --- | --- | --- |
| Fixed-duration easing | `EASE_OUT = [0.22, 1, 0.36, 1]` (JS tuple) ⇔ `--ease-out: cubic-bezier(0.22,1,0.36,1)` (`global.css`) | Reuse for every tween. Spring-driven items (B1/B2/B3) are exempt from `EASE_OUT` but stay transform/opacity-only. |
| Duration tokens | `--dur-fast 120ms`, `--dur-base 200ms`, `--dur-slow 360ms`, `--dur-rise 400ms` (`global.css` `@theme`) | Any fixed-duration timing maps to one of these; do not invent ad-hoc ms values. |
| Reduced-motion latch | Two existing patterns — the `prefersReducedMotion()` post-mount `matchMedia` function (reveal/terminal/control-section/argument-rail/team-preview/phrase-marquee) and Motion's `useReducedMotion()` (night-sky). | Each sub-item MUST degrade to its **final static state** using the latch **already present in its target component** — never introduce a third pattern. Verified per sub-item individually. |
| Perf envelope | — | Animate `transform` / `opacity` (and `filter` where already used, e.g. reveal blur) ONLY. GPU-composited, **CLS = 0**, zero layout-triggering properties (no width/height/top/left/margin transitions). |
| Amplitude | Brand voice: "barely noticed until removed" | Keep travel/scale/skew small and restrained; motion serves the content, never competes with it. |

> **`EASE_OUT` note for implementers:** there is no single exported `EASE_OUT` today — each client
> component re-declares the `[0.22, 1, 0.36, 1]` tuple locally (see `reveal.tsx` L9). Reuse the same
> tuple. Promoting it to one shared export from `@nightshift-ai/ui` is an acceptable (encouraged)
> refactor but not required; if promoted, migrate existing re-declarations in the same change.

## Tier A — Narrative scroll

### A1 — Moon Y-arc + dawn backdrop hue shift

- **Target:** `packages/ui/src/lib/nightshift/night-sky.tsx` (the `NightSky` component, rendered once
  per page as the fixed background — `variant="home"` on the home page, `apps/marketing/src/app/(frontend)/page.tsx` L68).
- **Technique:** Add page-scroll linkage via `useScroll()` (whole-document `scrollYProgress`, 0→1).
  Map progress through `useTransform` to the crescent-moon wrapper's `y` (top-of-page rise → set near
  `#install`) — compose with the existing `midX/midY` pointer-parallax transforms on the moon
  `motion.div` (night-sky L481–498) rather than replacing them. Drive a sibling full-bleed backdrop
  `motion.div` overlay whose `opacity` (transform/opacity-safe) ramps a `--terra-glow`/pre-dawn
  gradient in as progress approaches the `FinalCta` band. The existing 20s float
  (`y: [0,-22,0]`) stays as an additive idle drift.
- **Tokens/constants:** moon start geometry already at `top:178, left:'84%'` (do not animate those —
  drive `y` transform only); dawn glow uses `--terra-glow` / `--terra-tint`; existing `SPRING` may
  smooth the scroll value. Use `--ease-out` only if a discrete keyframe is added; the scroll-linked
  transform itself is continuous (no easing token needed).
- **Reduced-motion fallback:** `NightSky` already reads Motion's `useReducedMotion()` (`animate` flag,
  L260–261). When reduced, bind the moon to its static resting position (arc end-state) and render the
  dawn overlay at its final opacity — no scroll-linked movement.
- **Perf:** `transform` (`y`) + `opacity` only; the moon and overlay are already `position:absolute`
  in a fixed layer — zero reflow, CLS = 0.

### A2 — HowItWorks 5-stage scroll advance

- **Target:** `apps/marketing/src/components/home/how-it-works.tsx` (section `#how-it-works`, the
  `PIPELINE` strip, L135–196). Currently `status` (`done|active|idle`) is **hardcoded** per stage.
- **Technique:** Convert the strip to a client island (its own `'use client'` sub-component, keeping
  the surrounding server component intact). Use `useScroll({ target, offset })` scoped to the strip and
  `useTransform` to derive the active-stage index from scroll progress crossing each stage threshold
  (5 equal bands). The active stage receives the existing `STAGE_STYLE.active` treatment
  (`--border-accent` border, `--terra-400` command, `--glow-accent` box-shadow) via animated
  `opacity`/box-shadow-glow; passed stages take the `done` treatment.
- **Tokens/constants:** reuse `STAGE_STYLE` (L72–83) verbatim — `--glow-accent`, `--border-accent`,
  `--terra-400`, `--success`; `EASE_OUT` + `--dur-base` for the indicator crossfade between stages.
- **Reduced-motion fallback:** use the `prefersReducedMotion()` function pattern. When reduced, render
  the strip in its final/default stage state (the current hardcoded static composition) with no
  scroll-driven index change.
- **Perf:** animate `opacity` and the glow (box-shadow token swap) only; **do not** animate border-width
  or size. Active-stage indicator movement is opacity crossfade, not layout. CLS = 0.

### A3 — TeamPreview dot→charter shared-element morph

- **Target:** `apps/marketing/src/components/home/team-preview.tsx` (section `#agents`). The org tree
  (`TREE_ROWS`, L308+) sets `active` on hover; `getSidePanel(active)` renders the charter panel
  (right column, `lg:grid-cols-[1fr_350px]`).
- **Technique:** Give each agent tree dot (`motion.span`, `data-testid="team-dot"`, L344) and its name
  a shared `layoutId` (keyed by `row.key`) that matches an element in the charter-panel header, so
  Motion runs a shared-element (`layout`/`layoutId`) morph when `active` changes. Wrap the charter
  panel **body** in `AnimatePresence` (`mode="wait"`) keyed on `active` so it enters/exits on selection
  change. Keep the existing `active` state machine and `onMouseLeave` reset.
- **Tokens/constants:** dot colours already token-driven (`TONE_COLOR`, `--terra-400` active); panel
  enter/exit uses `EASE_OUT` + `--dur-base`; the twinkle loop (`TWINKLE_ANIMATE`) is untouched.
- **Reduced-motion fallback:** use the component's existing `prefersReducedMotion()` latch
  (`reducedMotion` state, L233/240). When reduced, disable the `layout` morph (render the panel header
  in place) and switch `AnimatePresence` content instantly (`initial={false}`, no enter/exit motion) —
  the panel still updates on hover, just without animation.
- **Perf:** `layout`/`layoutId` morphs compile to `transform`; panel body transitions are
  `opacity`/`transform`. The 350px panel column is fixed-width in the grid — the morph must not resize
  the grid track (CLS = 0). Restrain: subtle, not a flourish.

## Tier B — Micro-interaction

### B1 — Primary buttons pull toward pointer

- **Target:** the **Install** action (the `InstallSnippet` copy button — `CtaButton variant="primary" size="sm"`,
  `packages/ui/src/lib/nightshift/install-snippet.tsx` L71) and the **★ Star on GitHub** button
  (`CtaButton variant="secondary"`, in `hero.tsx` L91 and `final-cta.tsx` L86).
- **Technique:** `CtaButton` is a pure server component (`button.tsx`, CSS-only hover) — do **not**
  convert it wholesale. Add a thin **client** motion wrapper (or an opt-in `motion`-backed variant)
  that tracks pointer position over the button and drives a `useSpring`-smoothed `x`/`y` translate,
  pulling the button 4–6px toward the pointer on hover and releasing to `{x:0,y:0}` on
  `onPointerLeave`. Spring config **stiffness 300 / damping 20** (from the AC).
- **Tokens/constants:** spring params as specified (exempt from `EASE_OUT`); max pull amplitude 4–6px;
  no colour/glow change (the existing CSS `:hover`/`--glow-*` treatment stays).
- **Reduced-motion fallback:** `prefersReducedMotion()` function pattern in the wrapper — when reduced,
  render the button statically (no pointer-follow, `x/y` pinned to 0); CSS hover states still apply.
- **Perf:** `transform` translate only. Wrapper must not change layout box (CLS = 0).

### B2 — Hero + ProofBar settle spring on reveal

- **Target:** `apps/marketing/src/components/home/hero.tsx` and `proof-bar.tsx`, which currently reveal
  via `RevealGroup`/`Reveal` (tween, `duration` + `EASE_OUT`).
- **Technique:** Replace the tween transition on these two sections' reveals with a gentle settle
  spring (**stiffness 120 / damping 18**, from the AC) for the `opacity`+`y` entrance. Implement by
  extending `Reveal`/`RevealGroup` with an optional `spring`/`transition`-override prop (opt-in, keyed
  to a spring config) so the rest of the site keeps the tween — do NOT change the `Reveal` default.
  Hero and ProofBar opt in.
- **Tokens/constants:** spring params as specified (exempt from `EASE_OUT`); reveal travel stays the
  existing `Reveal` `y=16` default; keeps `viewport={{ once: true }}`.
- **Reduced-motion fallback:** already handled by `Reveal`'s identity-variants latch (`reveal.tsx`
  L142–148) — under reduced motion the item renders final/visible with no spring. The opt-in prop must
  preserve that latch.
- **Perf:** `opacity` + `transform` (`y`) only; CLS = 0.

### B3 — ControlSection controls tap-scale

- **Target:** `apps/marketing/src/components/home/control-section.tsx` — the **approve**, **cycle-mode**,
  and **± threshold** controls (reducer actions `INC_THRESHOLD` / `DEC_THRESHOLD` / `CYCLE_MODE` /
  gate-approve, L430–437).
- **Technique:** Add `whileTap={{ scale: 0.96 }}` to each of these interactive controls (render them as,
  or wrap them in, `motion` elements). No change to the reducer or control logic.
- **Tokens/constants:** scale target `0.96` (from AC); `whileTap` uses Motion's default tap transition
  (fast) — no `--dur-*` token needed; keep restrained.
- **Reduced-motion fallback:** component already uses the `prefersReducedMotion()` function pattern
  (L142). When reduced, omit `whileTap` (controls remain fully functional, no scale feedback).
- **Perf:** `transform` (`scale`) only; CLS = 0.

## Tier C — Terminal texture

### C1 — ProofBar per-digit roll

- **Target:** `apps/marketing/src/components/home/proof-bar.tsx` — the three stat numerals `11`, `10`,
  `60`, currently rendered by `CountUp` (`packages/ui/src/lib/nightshift/count-up.tsx`).
- **Technique:** On first viewport entry, roll each digit into place per-digit using a vertical
  `AnimatePresence` (each digit column translates `y` between old/new glyph, mono typeface). This
  supersedes or augments the current `CountUp` numeric tween for these three stats — implement as a
  `RollingNumber` client primitive (new, in `packages/ui/src/lib/nightshift/`) that ProofBar swaps in,
  keeping `CountUp` available for other uses. Trigger on IntersectionObserver like `CountUp` does.
- **Tokens/constants:** mono via existing `font-mono` class; accent digit colour `--accent` (as today,
  proof-bar L29/40/52); `EASE_OUT` + `--dur-base`/`--dur-slow` for each digit's vertical slide.
- **Reduced-motion fallback:** mirror `CountUp`'s three-layer degradation (`count-up.tsx` L49–52) —
  under reduced motion (or no IntersectionObserver) render the final digits immediately, no roll.
- **Perf:** `transform` (`y`) + `opacity` inside a fixed-size digit box (overflow-hidden). Digit box
  width is reserved so rolling never reflows the row — CLS = 0.

### C2 — Gate-pass SVG checkmark draw

- **Target:** the gate-pass checkmark in `apps/marketing/src/components/home/control-section.tsx`
  (gate glyph `✓`/`⊘`, `buildCtrlGates`/`glyph` L263) **and** the `/why-sdlc` argument rail
  `apps/marketing/src/components/why-sdlc/argument-rail.tsx` (`GateNode`, renders `✓` when
  `state === 'passed'`, L209).
- **Technique:** Replace (or overlay) the passed-gate `✓` glyph with an inline SVG check path animated
  via Motion's `pathLength` `0 → 1` (`motion.path` / `motion.svg`) when a gate transitions to passed.
  Reuse a single shared `GateCheck` SVG primitive across both call sites.
- **Tokens/constants:** stroke colour `--success` (⇔ `--green-400`, argument-rail L84/136); draw
  duration `--dur-slow` (360ms) with `EASE_OUT`; keep alongside the existing `GATE_PULSE_ANIMATE`
  (do not remove the pulse).
- **Reduced-motion fallback:** both components use the `prefersReducedMotion()` function pattern
  (control-section L142, argument-rail L15/228). When reduced, render the check fully drawn
  (`pathLength: 1`) immediately — no draw animation.
- **Perf:** `pathLength` animates the stroke via `stroke-dashoffset`, GPU-safe, no layout. SVG box is
  fixed-size in place of the glyph — CLS = 0.

### C3 — Scroll-triggered conventional-commit typewriter

- **Target:** a conventional-commit example rendered near `how-it-works.tsx` / `final-cta.tsx`. Reuse
  `packages/ui/src/lib/nightshift/terminal.tsx`, which **already** implements a character-reveal
  typewriter with a blinking caret (`CARET_BLINK_S = 0.5`, L66; char-reveal + `isCaret` logic
  L468–487).
- **Technique:** Render a `Terminal` instance (scripted-lines mode) with a conventional-commit example
  line (e.g. `feat(auto): …`), trigger the type-on when it scrolls into view (IntersectionObserver, as
  the terminal's existing reveal path does). Do **not** build a new typewriter — configure the existing
  Terminal. The page **H1 is explicitly excluded** from any typewriter treatment (out of scope).
- **Tokens/constants:** caret blink `CARET_BLINK_S` (existing); mono typeface + terminal surface
  (`--surface-terminal`) already in Terminal; commit-token colours reuse `--code-*` tokens.
- **Reduced-motion fallback:** Terminal already gates on `prefersReducedMotion()` (L126/140/252/282)
  and renders the full text statically with no caret when reduced — inherit that behaviour.
- **Perf:** character reveal is `opacity`/`transform` per-span within a reserved text block; the caret
  is opacity-only. Reserve final text height so reveal doesn't reflow — CLS = 0.

## Tier D — Delight / cohesion

### D1 — PhraseMarquee velocity-responsive speed/skew

- **Target:** `apps/marketing/src/components/home/phrase-marquee.tsx` — the looping mono ticker
  (`animate(x, ['0%','-50%'], { ease:'linear', repeat:Infinity })`, L51–55; hover-pause via
  `controlsRef`).
- **Technique:** Subtly modulate the marquee in response to scroll velocity: read `useScroll()` →
  `useVelocity(scrollYProgress)`, map (via `useTransform`, clamped) to a small `skewX` and/or a
  transient speed nudge on the track. Keep the base 34s linear loop and the hover-pause intact — the
  velocity response is an additive, restrained overlay that settles back to rest.
- **Tokens/constants:** base duration `MARQUEE_DURATION_S = 34` (existing); skew amplitude small
  (single-digit degrees) and clamped; band colours (`--terra-500`) unchanged.
- **Reduced-motion fallback:** component uses the `prefersReducedMotion()` function pattern (L16/45–47)
  and already renders a static band when reduced — under reduced motion, skip the velocity linkage
  entirely (no skew, no speed change).
- **Perf:** `transform` (`skewX`, translate) only; the marquee is already `overflow-hidden` and
  `aria-hidden` — CLS = 0.

### D2 — Draggable agent constellation (hover/tab-gated easter egg)

- **Target:** `apps/marketing/src/components/home/team-preview.tsx` (`#agents`), reusing the 11-agent
  data (`agents`, `team-data.ts`). Presentationally aligned with `NightSky`'s star aesthetic.
- **Technique:** Render the 11 agents as draggable star nodes within a **constrained** star-cluster
  layout using Motion `drag` with `dragConstraints` (bounded to the cluster container) and
  `dragElastic`. Discoverable **only on hover / keyboard tab-focus** — never a headline element, never
  auto-presented. This stays a non-headline easter egg (explicit out-of-scope: promoting it to a
  marketing element).
- **Tokens/constants:** star colours reuse `TONE_COLOR` / `--terra-400` / `--moon-100`; drag uses
  Motion spring defaults; keep amplitude and prominence low.
- **Reduced-motion fallback:** `team-preview.tsx` `prefersReducedMotion()` latch. When reduced, render
  the constellation static and non-animated; dragging (a direct-manipulation affordance) may remain
  available but without spring/inertia — no ambient motion.
- **Perf:** `transform` (drag translate) only; nodes are absolutely positioned in a fixed-size cluster
  box — CLS = 0. Keyboard focus target for tab-discoverability (a11y).

### D3 — Cross-page route transition

- **Target:** the four routes `/`, `/team`, `/why-sdlc`, `/faq` under
  `apps/marketing/src/app/(frontend)/`. There is currently **no** `template.tsx` (only `layout.tsx`,
  L36).
- **Technique:** Add a client `template.tsx` under `(frontend)/` wrapping `{children}` in
  `AnimatePresence` + a `motion.div` that cross-fades and rises (`opacity` + small `y`) on route
  change. `template.tsx` (not `layout.tsx`) is required so Next.js remounts the wrapper per navigation.
- **Tokens/constants:** `EASE_OUT` + `--dur-rise` (400ms) for the enter/rise; small `y` amplitude
  (consistent with `Reveal`'s `y=16`).
- **Reduced-motion fallback:** `prefersReducedMotion()` function pattern in the template — when reduced,
  render children with no transition (instant route swap).
- **Perf:** `opacity` + `transform` (`y`) only; the transition wraps page content without reserving/
  shifting layout — CLS = 0.

## Implementation sequencing (recommendation)

The four tiers are **independent** and can land in any order; this story is **not** split into per-tier
tickets (tracked as one deliverable per stakeholder direction — a future split may follow once
sequencing begins). Recommended order, lowest-risk / highest-leverage first:

1. **Tier B (micro-interaction)** — smallest, most isolated; B2 also establishes the opt-in
   spring-override on `Reveal` that nothing else depends on.
2. **Tier C (terminal texture)** — C1 `RollingNumber` and C2 `GateCheck` are new self-contained
   primitives; C3 only configures the existing Terminal.
3. **Tier A (narrative scroll)** — A1/A2/A3 touch shared/background components (`NightSky`,
   `HowItWorks`, `TeamPreview`) and carry the most scroll-linkage risk; sequence after the primitives
   exist (A3's morph pairs naturally with any TeamPreview work).
4. **Tier D (delight/cohesion)** — D3 route transition is a global wrapper best added once per-page
   motion is stable; D1/D2 are additive polish.

Each sub-item is independently demoable on its section (per AC), so tiers/sub-items may also be split
across multiple PRs if implementation prefers.

## Error handling

Motion-only story — no network/validation failure surface. Graceful-degradation rules:

| Scenario | Behaviour |
| --- | --- |
| `prefers-reduced-motion: reduce` active | Every sub-item (A1–D3) renders its **final static state**, verified per sub-item individually via that component's existing latch. |
| No `IntersectionObserver` (C1, C3, viewport-triggered) | Render final state immediately (mirror `CountUp`'s existing guard). |
| SSR / first-hydration frame | Deterministic static frame (matches existing night-sky/terminal/marquee pattern); motion starts post-mount only. |
| JS disabled | All content is server-rendered/visible; motion is pure enhancement — no content is gated behind it. |

## Out of Scope

- **Typewriter on the page H1** — explicitly excluded (per C3).
- **Any new animation dependency** beyond the motion.dev v12 install already in `packages/ui`.
- **Promoting the agent constellation (D2)** to a headline/marketing element — it stays a hover/tab-gated
  easter egg.
- **Any route or page beyond** `/`, `/team`, `/why-sdlc`, `/faq`.
- **Splitting this story into per-tier tickets now** — tracked as one deliverable; a future split pass
  may break Tiers A–D into separate stories.
- **Data/CMS/API/permissions/offline changes** — none (see Scope classification).
