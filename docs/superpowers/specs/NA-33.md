## Overview

Build the mid-page trust-building narrative of the nightshift home page — Home sections 4–7 (How it works, Review by day/ship by night, The team preview, Why builders choose it) — as a high-fidelity recreation of the approved design handoff, inside `apps/marketing`. This is the connective band between the hero conversion (NA-32) and the interactive control section (a separate story).

## Scope & applicable agents

- **web-engineer** — the only active agent this story touches (`apps/marketing/`, and reuse of primitives from `packages/ui/`).
- **No data model, no API, no CMS, no offline sync.** Per the Epic NA-29 CMS-vs-static constraint, all four sections are factual/reference content (how-it-works steps, agent roster/charters, workflow narrative, value propositions) and therefore stay **static in code**. No Payload collection/global, no endpoint, no migration, no seed change is introduced by this story.

Sections omitted as not applicable to this story: Data Model, API Surface, Backend Implementation, Mobile UI, Offline Sync.

## Web UI

All four sections live on the home route, composed into the existing home page after `ProblemSection`.

#### 3. `TeamPreview` — design L320–373 (`// 04 · the team`) — **client component**

- Section wrapper `id="agents"`. Full-bleed `bg-void` band.
- Centered header: `Eyebrow` (`04 · the team`), H2 `A team, not a megaprompt`, intro paragraph (verbatim L323–325).
- Two-column layout `grid-template-columns: 1fr 350px` (collapses to single column on narrow viewports):
  - **Left — terminal-tree box:** `surface-terminal` panel with the mono prompt line `$ nightshift --team --tree` (the `$` in `green-400`), then the box-drawing tree rows built from `team-data.ts`. Rows include: the human root (`you (human)`), each phase gate row (marked `⊘`, mono, with its slash-commands), and each agent row (twinkling star dot + name + note). Prefixes use box-drawing glyphs (`├──`, `└──`, `│`) exactly as the design composes them (L1112–1156).
  - **Right — sticky side panel** (`position: sticky; top: 110px`): shows the charter/contract for the currently-hovered row — eyebrow, initials chip, name, `STANDBY` tag where applicable, body (charter/`owns` or phase `contract`), optional `artifact →` line, and a repo reference link (`agents/<name>.md` or the repo root for the human/phase). Default (nothing hovered) state shows the org summary (`1 human · 11 agents` + the `⊘`/`✶` legend, verbatim L1176–1177).
- **Hover behaviour (AC3):** hovering a tree row (`onMouseEnter`) sets it as active → the side panel updates to that row's charter, and **all non-active rows dim** (`opacity: 0.45`; active row `opacity: 1`). `onMouseLeave` of the container clears the active row (panel returns to the org summary, all rows full opacity). State is local `useState<string | null>` (hovered row key) — **no global store**.
- Below the grid: the legend line (verbatim L370) and the **full-team-page link** (AC3): `Meet the whole team — charters, handoffs, org chart →` → `team.dc.html` in the design, which maps to the in-app route **`/team`** (the page built by the separate team-page story; the route already exists as a stub — link to `/team`).

**`team-data.ts` shape** — the static roster/charter data. Fully typed, no `any`:

```typescript
export type AgentTone = 'accent' | 'info' | 'cyan' | 'green';

export interface TeamAgent {
  /** Agent file name / tree label, e.g. 'solutions-architect'. */
  name: string;
  /** Charter shown as the side-panel body. */
  owns: string;
  /** Dot colour tone. */
  tone: AgentTone;
  /** True for roles not exercised on this repo (renders a STANDBY tag). */
  standby?: boolean;
}

export interface OrgPhase {
  /** Level id, e.g. 'L3'. */
  num: string;
  /** Phase label shown on the gate row. */
  phase: string;
  /** Slash-commands for this phase, e.g. ['/spec']. */
  commands: string[];
  /** Phase contract shown in the side panel when a gate row is hovered. */
  contract: string;
  /** Repo reference URL for the phase. */
  ref: string;
  /** Human-readable ref label. */
  refLabel: string;
  /** Agent names that belong to this phase. */
  agentNames: string[];
}
```

Populate `agents` (11 entries) verbatim from design L1361–1373 and `ORG_LEVELS` (L0–L6) verbatim from L679–723, including each phase's `contract`, `commands`, `ref`, `refLabel`, and the L5 `seqs` sequence notes. The `principal-engineer` orchestrates and the L5 domain agents (`database-administrator`, `platform-engineer`, `sync-engineer`, `web-engineer`, `mobile-engineer`) nest under it; `mobile-engineer` and `sync-engineer` are `standby`. Row/panel derivation logic mirrors design L1112–1182.

#### 4. `WhyDifferent` — design L376–401 (`// 05 · why different`)

- Centered header: `Eyebrow` (`05 · why different`), H2 `Why builders choose it`.
- 2×2 grid of four value cards (reuse `Card`; the design uses `border-color → border-accent` on hover, which `Card` already provides). Verbatim titles + bodies (L383–398):
  1. **The lifecycle is the product** — Spec → plan → implement → review, enforced by the commands and the handoff protocol. Tests are the merge gate. Review is done by a different agent than the one who wrote the code.
  2. **Generic agents, per-repo config** — No hardcoding. Every project fact lives in one `project-context.md`. The same plugin runs a Node monorepo, a Python service, or a mobile app.
  3. **Issue-tracker native** — It reads the ticket, derives the branch, plan, and PR, and comments the result back to Jira and GitHub.
  4. **Free, open, yours to fork** — Built on open Claude Code primitives, MIT-licensed. Fork it, extend it, swap a role. No lock-in, no paid tier gate.

## Error Handling

| Scenario                                            | Behaviour                                                                                                                             |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `InstallSnippet` clipboard API unavailable/blocked  | Already handled by the reused primitive — command stays visible/selectable, no throw, no error dialog                                 |
| `#workflow` deep-link target missing                | Prevented by contract: `DayNightWorkflow` MUST render `id="workflow"`; verified by AC2                                                |
| Full-team link target `/team` not yet built         | `/team` route stub already exists; link resolves. If the stub is absent, link still renders (no runtime error) — it is a plain anchor |
| JS disabled / hydration not yet run (`TeamPreview`) | Tree + default side panel (org summary) render server-side and are fully legible without hover; hover enhancement is progressive      |

**Reduced motion (AC5):** every interactive/hover/twinkle effect MUST degrade to a static, legible layout under `prefers-reduced-motion: reduce`:

- The tree's twinkle dot animation is gated behind `motion-safe` (no animation under reduced motion).
- Hover dimming/side-panel update remain functional but transition instantly (the global `transition-duration` guard in `global.css` already collapses transitions under reduced motion; reused `Card` already cancels its lift).
- The default, no-hover state of every section is fully legible on its own — reduced-motion users lose no information.
