# /sdlc:docs audit mode — Technical Spec

**Story:** NA-55 — Add /sdlc:docs audit mode
**Epic:** NA-50 — Public docs generation via /sdlc:docs command + knowledge-engineer
**Feature:** docs/features/2026-07-17-public-docs-generation-sdlc-docs.md
**Date:** 2026-07-18

## Overview

Add the fourth `/sdlc:docs` mode — `audit [--dry-run]` — which scans every activated doc-type row's
target path against the two source-of-truth surfaces the story names (accepted ADRs and current
code), reports the drift it finds, and — in its default (non-`--dry-run`) form — opens a PR carrying
the corrections it can derive **deterministically**. It reuses the command shell, the deterministic
regen algorithm (`refs/docs-pipeline.md` §3), the shared manifest gate, and the `knowledge-engineer`
agent shipped by NA-52/53/54; it adds no registry rows and edits no registry row.

## The shape decision — `audit` is drift-detect + deterministic-correct, **not** pure read-only, and **not** a founder-authoring mode

The task framing asked, up front, whether `audit` is a pure read-only report or a writing mode. The
story's acceptance criteria answer it (quoted inline below), and the answer drives which sibling
machinery is relevant and which is noise. Deciding it wrong in either direction would import the
wrong contract.

**AC2 is decisive: "Findings open a PR containing the corrections."** So `audit` **writes** — a
default run that finds drift opens a content PR. It is therefore **not** a pure read-only report. But
**AC3** ("`--dry-run` reports findings without opening a PR") carves out a read-only variant, and
**AC4** ("drift … is **flagged** as the doc diverging from the ADR") uses the verb *flag*, not
*correct*, for the ADR-vs-narrative case. The mode is thus **two-tier**, and the tiers map exactly
onto the Epic's existing hallucination-control split:

| Tier | Rows | What drift means | What `audit` does with it |
| ---- | ---- | ---------------- | ------------------------- |
| **Deterministic** | **every activated row with `generation-mode = auto`** | the published page differs from a byte-deterministic regeneration of its source-of-truth | **Corrects** it — the regenerated bytes are the correction, committed to the PR (un-gated, exactly as `sync` writes `auto` rows un-gated) |
| **Reference-integrity** | **every activated row with `generation-mode ∈ {draft-for-review, manual-only}`** | a **verifiable** anchor the page declares has moved — its `source:` globs changed, a referenced accepted ADR changed (AC4), or a referenced code path no longer exists | **Flags** it in the findings report — **never auto-rewrites the prose** (that would be fabrication) |

**The two `Rows` cells are defined by `generation-mode`, not by a hand-list — and the two sets must
partition the activated rows, so no row falls between the tiers unscanned (AC1).** The authoritative
keying is `generation-mode` (see the [Deterministic-correction tier](#deterministic-correction-tier-21--the-auto-rows)
and [Reference-integrity tier](#reference-integrity-tier-22--the-ac4-adr-direction-rule--the-narrative-rows)
sections below): every `auto` row is deterministic; **every non-`auto` row is
reference-integrity**. Resolved against `refs/doc-types.md` today — a **non-normative snapshot, not
the source of truth** (the registry is; resolve at read time) — the sets are: `auto` =
`command-reference`, `agent-reference`, `skill-reference`, `config-reference`, `hooks-contract`,
`error-reference`, `llms-txt` (7); reference-integrity = `changelog`, `release-notes`,
`migration-guide`, `tutorial`, `how-to`, `integration-guide`, `concept` (7). **`changelog` is
`draft-for-review`, so it is a reference-integrity row — never omit it:** a hand-list that drops it
leaves published-changelog drift unscanned, violating AC1's "scans every activated row". This is the
hardcoded-list drift class NA-54 hit (its unknown-type message once hardcoded the type set); the fix
is the same — derive from the `generation-mode` rule, treat any enumeration as an illustrative
snapshot.

**Why `audit` does not auto-rewrite narrative prose.** Rewriting a founder-authored concept doc to
"match" an ADR is exactly the hallucination the Epic's per-type gate model exists to prevent, and
AC4 says *flagged*, not *corrected*. Equally important, a narrative rewrite would not be a pure
function of source-of-truth, so `audit` could make no determinism claim about it (lessons from
NA-53/54). `audit` therefore corrects only what it can derive byte-deterministically and flags the
rest for the founder to fix with `seed`/manual editing — which already carry their own gates.

### What `audit` deliberately omits from the sibling machinery (and why)

Because `audit` writes **no** narrative content, it does **not** inherit the parts of the
`sync`/`release`/`seed` contract that exist solely to shepherd founder-facing narrative drafts:

- **No founder-confirm gate, and therefore no two-phase dispatch split.** The confirm gate exists in
  the three siblings for one reason: a dispatched subagent cannot pause for interactive input, so the
  gate must sit at the command layer *between* two dispatches (`refs/docs-pipeline.md` §2). `audit`
  presents nothing for confirmation — its `auto`-row corrections are un-gated by the Epic principle,
  and its narrative findings are flags, not writes — so it collapses to a **single**
  `knowledge-engineer` dispatch (scan → compute → in default mode, write + PR). This is the same
  un-gated path `sync` already takes when it resolves zero narrative drafts (§2: "the gate is skipped
  when there are zero narrative drafts — proceed straight to phase 2").
- **No topic/slug or version validation ladder.** `audit` takes no free-text argument that forms a
  path or ref — only the fixed literal flag `--dry-run` (see [Argument validation](#argument-validation)).
  So NA-53's `<version>` ladder and NA-54's `<topic>` slug ladder have **no analogue here**; importing
  a slug/version-style validation block would ship vacuous rules of exactly the kind NA-54 deleted and
  documented. Stated, and its absence justified, rather than copied by reflex.
- **No byte-identity/idempotence claim for a page body.** `audit` writes only regenerated `auto`-row
  pages, whose byte-identity is already covered by §3's determinism; it authors no page body of its
  own. The one determinism scope it **does** carry is stated precisely in
  [Determinism scoping](#determinism-scoping-scoped-and-load-bearing).

What it **keeps** from the siblings: the shared **manifest gate** (AC5 silent no-op), §3's
**deterministic regen algorithm** (to compute corrections), and the **release/seed branch/PR model**
— `Audit-Generated:` trailer, local-branch precondition, both re-run guards, never reset/force-push
(§13/§18) — because, like `release`/`seed`, `audit` drives `/loop /sdlc:loop` against its PR and so
its branch carries review-fix commits it must not clobber.

### Argument validation

Parse `$ARGUMENTS` into `<mode>` (first token) and the mode's remaining args. `audit`'s only argument
is the **optional, fixed-literal flag `--dry-run`**. There is **no free-text argument** — nothing that
forms a branch name, a path, or a ref — so **the slug/version-style validation ladders NA-53 and
NA-54 needed do not apply here, and adding one would ship vacuous rules of exactly the kind NA-54
deleted.** The entire validation surface is flag recognition, and its three cases were tested against
the shape of `$ARGUMENTS`:

1. **`audit` with no further tokens** → default (PR) mode. Not an error.
2. **`audit --dry-run`** (the flag is the sole remaining token) → dry-run (report-only) mode.
3. **`audit <anything-else>`** — any remaining token that is not exactly `--dry-run` (e.g.
   `audit --dryrun`, `audit foo`, `audit --dry-run extra`) → **usage STOP** with the usage message:

   ```text
   unknown argument for audit — usage: /sdlc:docs audit [--dry-run]
   ```

   A usage STOP (caller error), **distinct from the manifest-absent silent no-op** below. Rejecting an
   unrecognised token rather than silently ignoring it is the same discipline the siblings apply: a
   founder who fat-fingers `--dryrun` must not get a live PR-opening run they thought was a dry run.

**Scope note (v1):** `audit` takes **no** argument that narrows the scan to a single row or path — a
default run scans **every** activated row (AC1). A scoped/targeted audit is
[out of scope](#out-of-scope) and flagged as an [Open Question](#open-questions).

## `audit [--dry-run]` — behavioural contract

The full procedure is defined once in `refs/docs-pipeline.md` **§§20–24**; this command owns only the
manifest gate, the argument validation above, the single dispatch, and the report/PR surfacing. The
step order:

```text
1. argument validation  →  2. manifest gate (§1)  →  3. dispatch knowledge-engineer (audit)
→  4. scan + compute (§§20–22)  →  5a. --dry-run: report, no PR (§23)
                                    5b. default: write corrections + PR-on-change (§24)
```

1. **Argument validation** — the flag ladder above.

2. **Manifest gate (AC5).** Shared with `sync`/`release`/`seed` — defined **once** at
   `refs/docs-pipeline.md#manifest-gate-shared-by-sync-release-and-seed` (base-ref resolution
   pre-check → checkout-independent `git show` → silent no-op on genuine absence). **Not re-derived
   here**; `audit` is its fourth consumer. The shared section's title/anchor names only sync/release/
   seed today — **rename it to include `audit`** (a mechanical edit mirroring the rename NA-54 made
   when `seed` joined; update the header anchor and every pointer to it in `docs.md` and
   `docs-pipeline.md`). If **absent** → **silent no-op**: no scan, no dispatch, no PR, no error,
   **no stdout**, exit 0 (AC5). This holds for **both** `audit` and `audit --dry-run` — a repo that
   never opted in gets nothing, not even a "clean" report.

   The base-ref pre-check ordering is load-bearing for the reason §14 documents: a bare
   `git show origin/<BASE-BRANCH>:<path>` failure is ambiguous between "manifest genuinely absent"
   (the intended silent no-op) and "the **ref** does not resolve" (a fresh/shallow clone, a renamed
   base branch). Without the `git fetch` + `git rev-parse --verify origin/<BASE-BRANCH>^{commit}`
   pre-check first, an opted-in repo gets a silent no-op reporting success while auditing nothing.

3. **Dispatch `knowledge-engineer` (audit dispatch) — single dispatch, no confirm gate.** Pass it the
   run mode (`--dry-run` or default), the resolved activated-row set, and `origin/<BASE-BRANCH>`. The
   dispatch performs the scan (§§20–22), producing the [findings report](#findings-report-format).
   Because there is **no** founder-confirm gate ([shape decision](#what-audit-deliberately-omits-from-the-sibling-machinery-and-why)),
   this is a single dispatch, not a phase-1/phase-2 split.

4. **Scan + compute** (§§20–22): every activated row, split into the deterministic-correction tier and
   the reference-integrity flag tier.

5. **Report / PR:**
   - **`--dry-run` (§23):** the dispatch returns the findings report; the command **prints it to
     stdout** and opens **no** PR, writes nothing (AC3). A clean scan prints a one-line "docs are in
     sync — no drift found" (AC2's clean half still applies under `--dry-run`).
   - **default (§24):** if the scan produced **≥1 deterministic correction**, the dispatch writes the
     regenerated pages, embeds the full findings report (corrections **and** reference-integrity flags)
     in the PR body, and opens or updates the audit PR — **only if `git status --porcelain` on the
     written paths is non-empty**. A **clean scan** (no drift of either tier) opens nothing and reports
     clean (AC2). A scan that produced **only reference-integrity flags** (no deterministic correction
     to commit) reports them to stdout and opens **no** PR — there is nothing mechanical to commit and
     fabricating a narrative fix is forbidden (see the [Open Question](#open-questions) on report-only
     findings).

### Scan scope (AC1) — every activated row, orthogonal to `trigger`

`audit` is **not** keyed on the registry `trigger` column — **no row carries `audit` in its trigger**,
and none should. A default run scans **every row that is activated in the manifest** (present **and**
`enabled = true`, per the same "absent is never activated" rule §16 states for `seed` — never infer a
missing row as active), regardless of which mode *generates* it. This is `audit`'s distinct value:
`sync` is branch-diff-scoped (only rows a story touched) and `release`/`seed` are triggered by a
specific verb; `audit` is the **comprehensive** pass that catches drift the triggered modes never
revisit — a hand-edited reference page, a source file that changed without a `sync` run, an ADR that
moved after its concept doc was seeded.

**Every activated row must land in exactly one tier — the two tiers partition the activated set, keyed
solely on `generation-mode`.** There is no third bucket and no "unclassified" row: an `auto` row is
deterministic ([§21](#deterministic-correction-tier-21--the-auto-rows)), and **every** non-`auto` row
(`draft-for-review` **or** `manual-only`) is reference-integrity
([§22](#reference-integrity-tier-22--the-ac4-adr-direction-rule--the-narrative-rows)). An implementer
who builds the scan from a hand-copied row list instead of this rule reintroduces exactly the
`changelog`-omission defect the [shape-decision table](#the-shape-decision--audit-is-drift-detect--deterministic-correct-not-pure-read-only-and-not-a-founder-authoring-mode)
warns against.

Row activation is resolved from the manifest **checkout-independently** at `origin/<BASE-BRANCH>`
(the manifest gate already fetched and validated the base ref). `refs/doc-types.md` unreadable or
malformed → **surface and STOP**; never audit against a partial registry, and never fall back to a
hardcoded row list (the same discipline `seed`'s `SEED_TYPES` resolution applies).

### Deterministic-correction tier (§21) — the `auto` rows

For each activated row whose `generation-mode` is `auto`, `audit` reuses **§3's deterministic regen
algorithm** to regenerate the row's pages from **current** source-of-truth (at `origin/<BASE-BRANCH>`),
then compares the regenerated bytes against the **published** page at the same base ref
(checkout-independently — never the working tree, for the same reason every sibling reads that way):

- **A non-empty diff IS drift**, provably — the published page is not what its source-of-truth would
  currently generate. **The correction is the regenerated bytes.** No semantic judgment is involved.
- **Affectedness keying does not apply.** §3's "affected when `CHANGED_FILES` contains …" columns are
  a `sync`-only optimisation over a branch diff. `audit` has no branch diff; it treats **every**
  activated `auto` row as in-scope and regenerates it unconditionally. `audit` reuses §3's per-row
  *regeneration procedure*, not its *affectedness keying*.
- **`llms.txt` is an `auto` row and is regenerated like the others** (§8's format) — but see §24: a
  drifted `llms.txt` is itself a correction, and its regen also reflects any pages this run corrected.

Every deterministic correction is a pure function of source-of-truth (byte-reproducible), which is
what lets it be committed **un-gated** — the same "auto rows are un-gated" discipline `sync`,
`release`, and `seed` all apply.

### Reference-integrity tier (§22) + the AC4 ADR-direction rule — the narrative rows

For each activated **narrative** row (`generation-mode` ∈ {`draft-for-review`, `manual-only`} — which
includes `changelog`, `release-notes`, `migration-guide`, `tutorial`, `how-to`, `integration-guide`,
`concept`), `audit` **does not re-judge the prose** and **does not rewrite it.** It checks only
**verifiable anchors the page itself declares**, each a deterministic git/filesystem signal, and
**flags** (never corrects) a hit:

- **`source:`-glob drift (`how-to`, `integration-guide`).** A page carrying `source:` frontmatter
  (§5) whose globbed source files have commits **newer than the page's own last commit** is flagged as
  potentially stale. This is the same signal `sync` uses to draft a refresh — but `audit` surfaces
  pages `sync`'s branch-scoping never reached. A `source:`-less page is **not** flagged (§5's opt-in
  boundary: absence of `source:` means "never auto-tracked", and `audit` respects it).
- **Referenced-ADR drift (AC4).** A page that **declares an ADR reference** (an inline link to
  `docs/adr/NNNN-*` or an explicit ADR-reference frontmatter key) whose referenced ADR has commits
  newer than the page is flagged **as "the doc diverges from the ADR"** — the ADR is source of truth,
  **the direction is fixed, and `audit` never proposes the reverse**: it reads `docs/adr/**`
  read-only and **never writes `docs/adr/`** (the `adr` row is `public: no` — `audit` is a
  *public-docs* accuracy tool). A page with **no** declared ADR reference is **not** attributed to an
  ADR — `audit` never fabricates the association (the no-fabrication discipline NA-53's ADR-link rule
  applies). **This is where the [OQ carry-over](#oq-carry-over--audit-checks-for-frontmatter-the-templates-do-not-emit)
  bites:** the current templates emit no ADR-reference key, so this flag fires only for pages that
  carry one by convention — an [Open Question](#open-questions).
- **Dangling code reference.** A page that references a repo-relative code path which no longer exists
  at the base ref is flagged. Deterministic (path existence check), never a semantic guess.

**Pure narrative divergence with no verifiable anchor is deliberately NOT audited in v1** — detecting
that hand-authored prose has semantically drifted from an ADR requires an LLM judgment that is neither
deterministic nor reproducible, and acting on it would be fabrication. `audit`'s narrative tier
reports only anchor-backed signals, and says so.

### Findings report format

The report is a structured, deterministic enumeration (Markdown), grouped into the two tiers, one
entry per finding: `<row-type> · <page path> · <drift kind> · <evidence>` (e.g.
`command-reference · docs/reference/commands/docs.md · regen-diff · 12 lines differ from current
frontmatter+body`; `concept · docs/concepts/offline-sync.md · adr-drift · docs/adr/0007-*.md changed
2 commits after page`). It is emitted to stdout in **both** modes and, in default mode, embedded
verbatim in the PR body so a reviewer sees every flag — including the narrative flags `audit` did not
auto-correct — alongside the committed corrections.

### Determinism scoping (scoped and load-bearing)

`audit` carries **one** determinism claim, and it is scoped exactly:

> The **deterministic-correction set** (the `auto`-row regenerated pages and `llms.txt`) is a pure
> function of source-of-truth at the base ref: re-running `audit` over an unchanged repo produces the
> **byte-identical** correction set, so a re-run with no new drift commits nothing and opens no PR.

It makes **no** determinism claim about the **reference-integrity flags' phrasing or completeness** —
those are derived from verifiable git/filesystem signals (deterministic *inputs*) but are surfaced as
human-facing prose findings, and `audit` does not assert their wording is byte-reproducible. And it
makes **no** claim of detecting semantic narrative drift at all. This precise scoping is the direct
application of the NA-53/54 lesson: state only the determinism that actually holds.

### Branch / PR / control flow (§24)

`audit`'s corrections are fully derived, but — like `release`/`seed` — it drives `/loop /sdlc:loop`
against its PR, so the branch carries review-fix commits and **must not be reset or force-pushed.** It
adopts the release/seed model, not `sync`'s reset model:

- **Branch** `docs/audit-<YYYY-MM-DD>`, cut from `origin/<BASE-BRANCH>` head — mirroring
  `commands/adr.md`'s `docs/adr-distill-<YYYY-MM-DD>` dated-branch convention for a periodic scan (a
  same-day re-run open-or-updates the same branch/PR). The date forms the **branch name only** — it
  never feeds a written byte, so it does not compromise the determinism claim above. (Dated vs
  evergreen `docs/audit` is a minor [Open Question](#open-questions).)
- **Commit / PR title** `docs(docs): audit <YYYY-MM-DD>`, via `conventional-commit`, carrying the
  trailer **`Audit-Generated: <YYYY-MM-DD>`** — load-bearing, keyed on by both re-run guards.
- **PR base** `<BASE-BRANCH>` from project-context (never assume `main`).
- **Local-branch precondition, both re-run guards, and the reset/force-push prohibition** are adopted
  from §13/§18 verbatim in spirit (retargeted to the `Audit-Generated:` trailer and the audit branch
  name). The re-run content guard's scanned path set is **every path this run writes** — the corrected
  `auto`-row pages **plus** `llms.txt` — never a narrower set, for the reason §13/§18 give.
- **Write + PR only on change (AC2/AC4-parity):** commit and open-or-update the PR only if
  `git status --porcelain` on the written paths is non-empty; otherwise no commit, no PR, report clean.
- **Control-flow tail:** after a PR is raised, drive `/loop /sdlc:loop <PR_URL>` (fallback
  `ScheduleWakeup`), exactly as the three siblings do. If the run ended **before** any PR (manifest-
  absent silent no-op, a clean scan, a reference-integrity-flags-only scan, a usage STOP, or a
  precondition/guard STOP), release directly via `scripts/session-complete.sh` — only the manifest-
  absent path is silent. **`--dry-run` always ends before a PR** and releases this way.

## Error Handling

| Scenario | Behaviour |
| -------- | --------- |
| `.claude/project/docs-manifest.md` absent | **Silent** no-op — no scan, no dispatch, no PR, no error, **no stdout** (AC5), for both `audit` and `audit --dry-run`. Distinct from a usage STOP, which prints. |
| `audit` with an unrecognised argument (`--dryrun`, `foo`, `--dry-run extra`) | Usage STOP — `unknown argument for audit — usage: /sdlc:docs audit [--dry-run]`. Never silently treated as a plain `audit`. |
| Recognised future-mode token (`distill` only, after the strike) | Print "mode not yet implemented (see Epic NA-50)" and exit cleanly — not an error. **`audit` is no longer in this set.** |
| `git fetch` fails, or `origin/<BASE-BRANCH>` will not resolve | Surface and STOP — resolved by the manifest gate's base-ref pre-check **before** the absence conclusion; never mistaken for "manifest absent". |
| `refs/doc-types.md` unreadable/malformed | Surface and STOP — never audit against a partial registry, never fall back to a hardcoded row list. |
| Manifest present, **no** row activated | Clean scan — report "no activated doc types to audit"; no PR. Informational, not silent (the manifest exists). |
| `--dry-run`, drift found | Print the findings report; **no** write, **no** PR (AC3). |
| `--dry-run`, no drift | Print "docs are in sync — no drift found"; no PR. |
| default, ≥1 deterministic correction | Write the regenerated `auto` pages, embed the full findings report in the PR body, open-or-update the PR (AC2). |
| default, drift is **only** reference-integrity flags (no deterministic correction) | Report the flags to stdout; **no PR** — nothing mechanical to commit, and a narrative correction is never fabricated. (See the report-only [Open Question](#open-questions).) |
| default, no drift of either tier | Report clean; open nothing (AC2). |
| A narrative page diverges from an ADR it references | **Flagged** as "the doc diverges from the ADR" (AC4) — ADR is source of truth, direction fixed. **Never** written into `docs/adr/`, never the reverse. |
| A narrative page references no ADR | Not attributed to any ADR — the association is never fabricated. |
| A `source:`-less how-to page | **Not** flagged for `source:`-glob drift — §5's opt-in boundary is respected. |
| Deterministic regen produced byte-identical output for every `auto` row | No commit, no PR (empty `git status --porcelain`) — the determinism-scoped no-op. |
| Re-run: audit branch carries **out-of-pipeline** edits to generated pages (no `Audit-Generated:` trailer) | **STOP** (re-run content guard) — surface the paths and the PR number; never overwritten. |
| A **local** `docs/audit-<DATE>` holds unpushed commits | **STOP** (local-branch precondition) — never discard unpushed work, never `checkout -B` over it. |
| `gh` / `raise-pr.sh` failure | STOP and surface — the write is on a branch and reviewable; never leave a raised-but-broken state silently. |
| claude-mem tools unavailable | No effect — `audit` consults no corpus. |

## Out of Scope

- **Auto-rewriting narrative prose** to match an ADR, code, or any source-of-truth — flagged only,
  never fabricated (the shape decision).
- **Semantic narrative-drift detection** (LLM judgment that hand-authored prose has drifted from an
  ADR without a declared anchor) — v1 flags only anchor-backed, verifiable signals.
- **Writing `docs/adr/`** — `audit` reads accepted ADRs read-only; producing/distilling ADRs is NA-57.
- **Scoped/targeted audits** (auditing a single row, path, or quadrant) — v1 scans every activated
  row (AC1).
- **Auto-triggering on CI / tag-push events** — explicit Epic out-of-scope; `audit` is founder-invoked.
- **`sync`/`release`/`seed`/`distill` modes** and **post-merge/dual-diff-source sync** (that is NA-56).

### OQ carry-over — `audit` checks for frontmatter the templates do not emit

Filed as a standing gap across NA-53/54: the `writing-docs` templates emit **no `description:`
frontmatter** (three of four emit no frontmatter block at all). This gap is **directly in `audit`'s
path**, twice over:

1. `audit`'s deterministic tier regenerates the `auto` rows and its `llms.txt` regen (§8) derives each
   entry from page frontmatter — so a pre-existing `public: yes` page lacking `title`/`description`
   surfaces during `audit`'s `llms.txt` regen exactly as it does during `seed`'s (§19): **skipped and
   surfaced, never a STOP, never inferred**. `audit` **adopts §19's skip-and-surface rule verbatim** —
   it does not invent a new one, and it must not STOP the whole audit over one frontmatter-less page.
2. If `audit`'s narrative tier is ever extended to *check for* required frontmatter presence
   (`title`/`description`) as a drift signal, it would flag nearly every existing narrative page —
   which is a template-source defect, not per-page drift. **v1 does not add a frontmatter-presence
   flag** for this reason; it is recorded here so a later story fixes the templates at source rather
   than teaching `audit` to flag a gap the templates guarantee.

**Recommendation:** a follow-up story adds `description:` (and an ADR-reference key) to the
`writing-docs` templates — the single fix that unblocks AC4's reliable ADR join **and** removes the
frontmatter-less-page skip path across `sync`/`release`/`seed`/`audit`.
