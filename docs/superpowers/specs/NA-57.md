# Absorb /sdlc:adr into /sdlc:docs — Technical Spec

**Story:** NA-57 — Absorb /sdlc:adr into /sdlc:docs
**Epic:** NA-50 — Public docs generation via /sdlc:docs command + knowledge-engineer
**Feature:** docs/features/2026-07-17-public-docs-generation-sdlc-docs.md
**Date:** 2026-07-18

## Overview

Remove the standalone `/sdlc:adr` command and make `/sdlc:docs` the single surface for ADR curation: `/sdlc:docs seed adr "<pattern>"` seeds a founder-known ADR inline and `/sdlc:docs distill ["<focus>"]` distills the learnings corpus into promotable ADR candidates — both routing to the **retained** `refs/adr-pipeline.md` with the founder-confirm gate hoisted to the `/sdlc:docs` command layer. This is the **final** story of NA-50; after it, all five `/sdlc:docs` modes (`sync`/`release`/`seed`/`audit`/`distill`) are live and no standalone `/sdlc:adr` command remains.

## Scope of this story (authoring surface)

This is a **plugin-authoring** / **command-absorption** story. It adds **no new pipeline** and **no new `docs/**` data** — it re-routes an existing command surface (`/sdlc:adr`) onto an existing pipeline (`refs/adr-pipeline.md`) reached through an existing dispatch branch (the `knowledge-engineer` ADR dispatch). But a **full command removal touches everything the command was named in**, so the guiding constraint is not "minimise churn" — it is **completeness**: every reference to the removed command, and every behavioral guard that lived only in the removed file, must be accounted for or it silently regresses. All edits sit inside `plugins/sdlc/`, executed by the **ai-enablement-engineer** (owner of `plugins/`, `skills/`, `.agents/` per the project-context workspace→agent table).

### The two crux decisions, answered up front

1. **Absorb vs alias → full removal, no alias, no shim.** AC4 is explicit ("command file deleted, no alias/shim"); the Epic confirms "pre-1.0 plugin, founder-only user base — straight removal, no deprecation shim." `commands/adr.md` is **deleted**. The canonical surface becomes `/sdlc:docs seed adr "<pattern>"` and `/sdlc:docs distill ["<focus>"]`. There is **no** `/sdlc:adr` alias to keep resolving — so every doc/ref that names the command is rewired (AC5), and nothing is left to dangle at a removed command.

2. **What moves vs stays → the pipeline's *algorithm* stays, but the command surface it names re-points everywhere.** `refs/adr-pipeline.md` is **retained as the single-source pipeline** (AC1: "the retained `refs/adr-pipeline.md`") — its two-phase dispatch, distill evidence protocol, promotion criteria, numbering, supersede flow, and `docs/adr/index.md` regen are **unchanged as algorithms**. But **"retained" does not mean "untouched"**, and this spec explicitly rejects any "§§X–Y otherwise unchanged" framing — that framing is exactly what let the first draft leave live `/sdlc:adr` strings and command-layer guards in place. The file **names `/sdlc:adr` in live prose** (its §1 ownership note, its §2 concurrency example, its §5 claude-mem halt message) and those references re-point like every other. The edit set is **derived from greps, never enumerated by line number** — see [Deriving the removal edit-set](#deriving-the-removal-edit-set). Two things genuinely **relocate** out of the deleted `commands/adr.md` or they are silently lost: (a) the **command-layer wrapper** (founder-confirm gate flow, ADR branch/PR naming, control-flow tail), single-sourced into `refs/adr-pipeline.md` as a new "command-layer flow + branch/PR naming" section that `commands/docs.md`'s two routes point at (Standing lesson 4 — no duplicated normative prose); and (b) **two behavioral guards that live ONLY in `adr.md`, not in `adr-pipeline.md`** — the empty-argument STOP and the first-ADR base case — see [Behavioral guards that must relocate](#behavioral-guards-that-must-relocate). The `knowledge-engineer` ADR dispatch branch keeps its flow but its **selector logic changes** — it can no longer key on the command name (see [Executing-layer propagation](#executing-layer-propagation-standing-lesson-3)).

## Data Model

**Not applicable** — this story edits plugin Markdown (a deleted command, a retained pipeline ref re-pointed, a command shell re-routed, an agent's dispatch labels, and cross-reference text). No database entities, no migrations. ADRs continue to be written to `docs/adr/` exactly as the retained pipeline already does.

## API Surface

**Not applicable** — no HTTP endpoints. The "surface" here is the CLI command surface: `/sdlc:adr` is **removed**; `/sdlc:docs seed adr` and `/sdlc:docs distill` become live routing targets on the existing `/sdlc:docs` command.

## Command surface — before / after

| Old (removed)                   | New (canonical)                    | Notes                                                                                                              |
| ------------------------------- | ---------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `/sdlc:adr "<pattern>"`         | `/sdlc:docs seed adr "<pattern>"`  | Seed a founder-known ADR inline. Routes to `refs/adr-pipeline.md` (seed).                                          |
| `/sdlc:adr --distill`           | `/sdlc:docs distill`               | Distill the whole learnings corpus. Routes to `refs/adr-pipeline.md` (distill).                                   |
| `/sdlc:adr --distill "<focus>"` | `/sdlc:docs distill "<focus>"`     | Focus text becomes a **positional** after the `distill` mode token (the old `--distill` **flag** is gone).        |

**Note the argument-shape change for distill.** Old `/sdlc:adr` keyed distill off the `--distill` **flag** (the presence of the flag chose distill mode; seed was the flagless form). Under `/sdlc:docs`, `distill` is a first-class **mode token** (like `sync`/`release`/`seed`/`audit`), and the optional focus is a positional argument after it: `/sdlc:docs distill "QA review patterns"`. There is no `--distill` flag anywhere in the new surface.

## Routing decision (the core of this story)

### `seed adr "<pattern>"` — a special seed route, not a generic seed type

`adr` **stays excluded from `SEED_TYPES`** (`refs/docs-pipeline.md` §15 already excludes it; that exclusion is retained, not undone). The generic `seed` machinery (`docs-pipeline.md` §§15–19 — manifest gate, type-activation gate, `SEED_ROW` `target-path`, single narrative-page scaffold, page-exists gate) is **the wrong pipeline** for an ADR: an ADR is a numbered `docs/adr/NNNN-slug.md` file with `status: accepted`, a two-way supersede flow, and a deterministic `docs/adr/index.md` regen — none of which the generic single-page seed path does. So `seed adr` is a **special route**: the `commands/docs.md` seed handler branches on `<type> == adr` **before** the generic seed gates and dispatches the **ADR pipeline** (`refs/adr-pipeline.md`) instead.

This means the change from the current state is surgical: the argument-validation step that today prints the `seed adr … not yet implemented (see NA-57)` **clean stub** now instead **routes to the ADR pipeline**. `adr` never entering `SEED_TYPES` is preserved — it was always a special case, and it remains one; only the special-case *action* flips from "print stub, exit 0" to "dispatch adr-pipeline".

### `seed adr` and `distill` skip the docs-manifest gate — deliberately

The four generic `/sdlc:docs` modes gate on `.claude/project/docs-manifest.md` (silent no-op when absent) because they generate **public** doc types the repo opts into. **ADR is internal** (`refs/doc-types.md`: the `adr` row is `public = no`, `target-path = docs/adr/`) and the removed `/sdlc:adr` had **no manifest dependency** — it resolved `docs/adr/` from `project-context.md` and worked in any repo. Gating `seed adr` / `distill` on the manifest would be a **regression**: a repo with no `docs-manifest.md` (this SDLC repo has none today) would silently no-op ADR authoring that `/sdlc:adr` performed fine. Therefore:

- **`seed adr` and `distill` route straight to `refs/adr-pipeline.md` with NO manifest gate and NO type-activation gate.** The `docs/adr/` target is resolved from `project-context.md` exactly as the retained pipeline already does. This preserves the removed command's behaviour intact — the command is gone, the behaviour is not.
- The `adr` registry row in `refs/doc-types.md` (`trigger = seed,distill`, `public = no`, `target-path = docs/adr/`) is **confirmed unchanged** and satisfies **AC3** (ADRs are never published to public docs). No edit to the row's values is required; the spec only asserts the invariant holds. (`refs/doc-types.md`'s registry self-check already mandates `adr` → `public = no`, `target-path = docs/adr/`.)

### `distill` — from stub to live mode

`distill` is today a **future-mode stub** in `commands/docs.md` (argument-validation step 3 prints `mode "distill" is not yet implemented (see Epic NA-50)`). AC2 makes it **live**. The `distill` mode:

- Takes an **optional positional focus** (`/sdlc:docs distill ["<focus>"]`), passed through to the ADR pipeline's distill evidence protocol exactly as the old `--distill "<focus>"` did.
- Runs the ADR pipeline's **two-phase dispatch** (`refs/adr-pipeline.md` §2): Phase 1 mines the corpus and returns candidates + per-candidate deletion lists; the **founder-confirm gate** at the `commands/docs.md` layer presents them; Phase 2 writes accepted ADRs (`status: accepted`), deletes founder-approved learnings in the same PR, and regenerates `docs/adr/index.md`.
- **No manifest gate** (as above).

### After this story, the future-mode stub set is empty — the reachability precondition

Every prior mode story hit this: a mode's own stub must be **struck** when the mode goes live, or a live routing target still prints "not yet implemented". This story makes `distill` **and** `seed adr` live — the **last** two stubs on the surface. After the strikes:

- `commands/docs.md` argument-validation **step 3** ("Recognised future-mode token (`distill`)") has **no remaining member** — the recognised-but-future tier is now empty. It is removed (leaving the unrecognised-token STOP and the live-mode routes). A future edit must not treat the empty tier as a bug to "restore".
- `commands/docs.md` **step 8** (the `seed adr` clean stub) is struck and replaced by the `seed adr → adr-pipeline` route.
- `commands/docs.md` **error-handling row** for the recognised-future-mode token (`Print "mode not yet implemented (see Epic NA-50)"`) is struck — after this story there is **no** recognised-but-future token, so a row that still shadows the now-live `distill` with a stub is itself the reachability regression. This row contains **no `/sdlc:adr` string**, so the AC5 command grep is structurally blind to it — see [Deriving the removal edit-set](#deriving-the-removal-edit-set) for the phrase greps that catch it.
- The `refs/doc-types.md` note "Only `distill` (NA-57) remains specified-but-not-yet-built" is struck — nothing is unbuilt after this story.

## Behavioral guards that must relocate

Two guards existed **only** in the now-deleted `commands/adr.md` (its command-layer flow and its error table). They are **absent from `refs/adr-pipeline.md`**, so deleting `adr.md` without relocating them is a silent correctness regression, not a cosmetic string miss. Each must land in its new home and be verifiable there.

1. **Empty-argument STOP → relocate to the command-layer `seed adr` handler in `commands/docs.md`.** `adr.md` STOPped on empty `$ARGUMENTS` ("no pattern") with a usage message, at the command layer. `adr-pipeline.md` has **no empty-pattern guard** — Phase 1 drafts from whatever pattern text it is handed. Without the relocated guard, `/sdlc:docs seed adr` or `/sdlc:docs seed adr ""` would dispatch Phase 1 with **empty input** and draft a **hallucinated ADR**, then raise a PR for it. The `seed adr` route in `commands/docs.md` MUST STOP with a usage message when the pattern text is empty, **before** dispatching Phase 1. Scope: this guards `seed adr`'s pattern only — `distill` with no focus is **valid** (it means "whole corpus"), so the empty-input STOP does **not** apply to `distill`.
2. **First-ADR base case → relocate into `refs/adr-pipeline.md`'s numbering step.** `adr.md`'s error table carried "`docs/adr/` does not exist yet (first ADR) → create it; next number is `0001`." `adr-pipeline.md` §2 step 5 / §10 only specify `NNNN = max(existing) + 1` and "list `docs/adr/`" — with **no** `mkdir` and **no** empty-set base. On the first-ever `/sdlc:docs seed adr` in a repo, `docs/adr/` does not exist, so `max(existing)` over an empty/absent directory is undefined and the write target directory is missing. `adr-pipeline.md`'s numbering step MUST state: if `docs/adr/` is absent, create it; an empty existing set bases `NNNN` at `0001`. This belongs in the pipeline (where numbering and writing live), not the command layer.

Both guards are **verifiable in the relocation target**: after the edits, `commands/docs.md`'s `seed adr` route contains the empty-pattern STOP, and `refs/adr-pipeline.md`'s numbering step contains the create-dir + `0001` base. If either target lacks its guard, the relocation is incomplete.

## Executing-layer propagation (Standing lesson 3)

The current dispatch-branch selector in `knowledge-engineer.md` disambiguates ADR vs docs dispatches by **which command dispatched the agent** ("via `/sdlc:adr`" vs "via `/sdlc:docs`"). After removal, **both** ADR and docs work arrive from `/sdlc:docs`, so the command name no longer disambiguates. The selector **must re-key on the mode+type**, or the absorbed `seed adr`/`distill` routing would reach the agent and load the wrong skill branch (the NA-56 lesson: a re-route is inert if the callee doesn't consume the new signal):

- **`seed adr` and `distill` → ADR dispatch** — load `writing-adrs`, run `refs/adr-pipeline.md`, write `docs/adr/**`. **Never** `writing-docs`.
- **`sync` / `release` / `seed <non-adr>` / `audit` → docs dispatch** — the existing docs-pipeline branches, unchanged.

`knowledge-engineer.md`'s existing tail already anticipates this: "a docs-sync, release, seed, or audit return must NOT list `writing-adrs` (this stays true even after NA-57 wires `seed adr`, which routes through the ADR dispatch branch, not this one)." The spec formalises it: **`seed adr` is classified as an ADR dispatch, not a generic seed dispatch** — it is the one `seed`-token invocation that loads `writing-adrs` and never `writing-docs`. The generic `seed dispatch` skill rules (Phase 1 `writing-docs`) apply to `seed concept|tutorial|integration-guide|how-to` only.

The command must pass the dispatch-type signal explicitly so the agent selects the ADR branch (there is no longer a command name to infer it from). The two-phase split, the founder-confirm gate at the command layer, and the ADR branch/PR naming are handed to the agent by `commands/docs.md`'s new routes exactly as `commands/adr.md` did — sourced from `refs/adr-pipeline.md`, not restated.

## Never collapse distinct failures (Standing lesson 2)

The absorbed pipeline carries two failure paths that must **not** be flattened into each other or into the generic seed path:

- **`distill` with claude-mem tools absent → HALT** (`refs/adr-pipeline.md` §5). The halt message is rewritten with **two** substring changes, not one: the command name `/sdlc:adr --distill` → `/sdlc:docs distill`, **and** the fallback `use seed mode` → `use seed adr` (there is no bare "seed" surface anymore — the ADR seed is `seed adr`). Final text: `claude-mem tools unavailable — /sdlc:docs distill requires the claude-mem plugin; install it or use seed adr`. This is a hard halt because the corpus is distill's **entire input**. It must **not** be softened into the generic **seed** dispatch's "proceed without corpus enrichment" behaviour — for `seed adr` (like every seed) the corpus is only enrichment, so absence is non-fatal there, but for `distill` it is fatal. The two are distinct and the spec keeps them distinct.
- **`distill` with no promotable candidates → clean no-op** (AC6): report "no candidates met the promotion criteria" and exit cleanly — no ADR, no PR. This is **distinct** from the tools-absent halt (a genuine environment failure) and from a git/`gh` failure (a STOP). An empty result is success-with-nothing-to-do; it never opens a PR and never errors.

## Reference auto-regen (Standing lesson 5)

**Derive the affected reference set from the file keys, not a hand-list.**

- **`command-reference`** is `commands/**`-path-keyed (`refs/docs-pipeline.md` §3). This story **deletes** `commands/adr.md` and **edits** `commands/docs.md` and `commands/analyze.md` — so a subsequent `/sdlc:docs sync` **removes** the generated `adr` command-reference page and **regenerates** the `docs` and `analyze` command-reference pages. All expected drift, not a bug. Do not hand-edit any generated page in this story.
- **`error-reference`** is content-keyed on the Error Handling tables across commands/agents/refs. `commands/adr.md`'s error table is deleted and `commands/docs.md`'s gains the `distill`/`seed adr` rows — so the generated `error-reference` page regenerates too. Expected.
- **Cross-links:** any generated reference page or `llms.txt` entry that linked the `adr` command page must resolve after regen — since the page is removed, the regen drops its entry. If a manually authored narrative doc (concept/how-to) linked `/sdlc:adr`, `audit`'s reference-integrity tier flags it; none exists in this repo today.
- **This repo has no `docs-manifest.md`**, so all of the above regen is a **silent no-op here** — the reference pages are not generated in this repo. The regen matters for downstream consumer repos that activate those rows. The spec notes it so an implementer does not expect generated-doc changes in this PR.

## Permissions Detail

Not a multi-role RBAC surface. The actors are the **founder** (invokes `/sdlc:docs seed adr` / `distill`, confirms drafts and deletions at the gate) and the **`knowledge-engineer` agent** (drafts in Phase 1, writes in Phase 2). The authority model is **unchanged from `/sdlc:adr`**: the founder-confirm gate remains the acceptance moment (drafts are `proposed`, confirmed writes are `accepted`), it stays at the **command layer** (now `commands/docs.md` instead of `commands/adr.md`), and distill's learning deletions remain founder-gated (sanctioned as Exception 2 in `analyze-protocol.md`). Removing the standalone command changes **where** the gate is hosted, not **who** authorises or **what** is gated.

## Out of Scope

- **Folding `refs/adr-pipeline.md` into `refs/docs-pipeline.md`.** The ADR pipeline is **retained as a separate single-source ref** (AC1). This story re-points its callers and rewires its own command-name references; it does not merge or abstract a shared pipeline (the epic's "copy the shape, do not abstract" rule).
- **Routing `seed adr` through the generic seed page-scaffold machinery** (`docs-pipeline.md` §§15–19). `adr` stays a special route to `adr-pipeline.md`; it does not become a generic `SEED_TYPES` member. An ADR is not a single narrative page.
- **Gating `seed adr` / `distill` on `docs-manifest.md`.** Deliberately not gated — ADR is internal and works in any repo, matching the removed command. Adding a manifest gate would regress ADR authoring in manifest-less repos.
- **Changing the `adr` registry row's values** in `refs/doc-types.md`. AC3 only **confirms** `public = no`; no value edit is needed.
- **Non-ADR `distill` targets** (troubleshooting / FAQ from the corpus) — explicitly a later Epic per the PRD/Epic open questions. `distill` here produces **ADR candidates only**.
- **Parked follow-ups — do not reopen** (this story closes NA-50; these belong to their own tickets): **NA-60** (`PROJECT_KEYS` derivation), **NA-61** (`writing-docs` template frontmatter emitting `title`/`description`), **NA-62** (the repo's non-idempotent Prettier on 2-space list-nested fenced blocks). This story only **flags** NA-62-shaped blocks it edits (artifact table) for the implementer to dedent; it does not attempt the Prettier fix.
- **Auto-triggering ADR distill on any event** (CI, tag, merge) — epic out-of-scope; the trigger stays the founder-invoked command.

## Open Questions

- [ ] **Home for the relocated command-layer flow + ADR branch/PR naming.** `commands/adr.md`'s founder-confirm-gate flow, branch names (`docs/adr-<slug>`, `docs/adr-distill-<YYYY-MM-DD>`), PR titles (`docs(adr): …`), and control-flow tail must live somewhere after the file is deleted. _Suggested default (adopt):_ **relocate into `refs/adr-pipeline.md`** as a new "command-layer flow + branch/PR naming" section, single-sourced, with `commands/docs.md`'s two new routes as thin pointers — mirroring how the four docs modes point at `refs/docs-pipeline.md`. _Alternative (rejected):_ inline the flow into `commands/docs.md` — duplicates normative prose the two ADR modes and the retained pipeline must agree on (Standing lesson 4), and bloats the command shell.
- [ ] **`seed adr`/`distill` PR scope — `docs(adr):` vs `docs(docs):`.** The generic docs modes commit as `docs(docs): …`; the removed ADR command used `docs(adr): …`. _Suggested default (adopt):_ **keep `docs(adr):`** for both ADR routes (branch `docs/adr-<slug>` / `docs/adr-distill-<date>`, PR title `docs(adr): …`) — the ADR pipeline's numbered-record flow and `writing-adrs` conventions are distinct from public-doc generation, and the retained-pipeline principle argues for keeping its established naming. _Alternative:_ unify under `docs(docs):` for surface consistency — rejected as churn that muddies ADR PRs with public-docs PRs in `git log`/changelog aggregation.
- [ ] **`distill` focus argument quoting.** The old `--distill "<focus>"` took a quoted focus after the flag; `distill "<focus>"` takes it as a positional. _Suggested default (adopt):_ treat everything after the `distill` mode token as the focus text (same as the old post-`--distill` text), no additional parsing — an empty positional means "whole corpus". No flag form is accepted. Low risk; noted for the implementer so the parse matches the old semantics exactly.
