# /sdlc:docs command + sync mode — Technical Spec

**Story:** NA-52 — Add /sdlc:docs command + sync mode
**Epic:** NA-50 — Public docs generation via /sdlc:docs command + knowledge-engineer
**Feature:** docs/features/2026-07-17-public-docs-generation-sdlc-docs.md

## Overview

Ship the `/sdlc:docs` command shell, `refs/docs-pipeline.md` (mirroring `refs/adr-pipeline.md`'s shape), and the first mode — `sync <STORY-KEY>` — which diff-drives deterministic regeneration of frontmatter-driven reference docs plus `llms.txt`, and drafts gated how-to refreshes for the narrative rows a diff touches. This is the deterministic foundation the other four `/sdlc:docs` modes (release / seed / audit / distill) build on.

## Scope of this story (authoring surface)

This is a **plugin-authoring** story. It ships three edits inside `plugins/sdlc/`, executed by the **ai-enablement-engineer** (owner of `plugins/`, `skills/`, `.agents/` per the project-context workspace→agent table). The runtime agent that `sync` dispatches — **knowledge-engineer** — is extended, not created.

## Command Contract — `/sdlc:docs`

### Mode table (v1 ships only `sync`)

| Invocation | Mode | v1 status |
| --- | --- | --- |
| `/sdlc:docs sync <STORY-KEY>` | **sync** | **In scope — this story** |
| `/sdlc:docs release <version>` | release | Out of scope (NA-53) — the command surfaces a "not yet implemented" stub, not an error |
| `/sdlc:docs seed <type> [topic]` | seed | Out of scope (NA-54/55) — stub |
| `/sdlc:docs audit [--dry-run]` | audit | Out of scope (NA-54/55) — stub |
| `/sdlc:docs distill` | distill | Out of scope (NA-56) — stub |

- Empty `$ARGUMENTS` or an unrecognised first token → STOP with a usage message:
  `usage: /sdlc:docs sync <STORY-KEY>  (release|seed|audit|distill land in later stories)`.
- A future-mode token (`release`/`seed`/`audit`/`distill`) → print `mode "<mode>" is not yet implemented (see Epic NA-50)` and exit cleanly. Not an error, not a STOP with non-zero intent — a deliberate stub so the surface is stable before the modes land.

### `sync <STORY-KEY>` — behavioural contract

1. **Manifest gate (AC5).** Resolve `.claude/project/docs-manifest.md`. If **absent** → silent no-op: no branch, no dispatch, no PR, no error, no stdout beyond exiting cleanly. This is the zero-setup-cost guarantee for repos that declined the `/init` docs opt-in.
2. **Diff source (v1, single source).** Compute the changed-file set for the story as `git diff --name-only <BASE-BRANCH>...HEAD`, where `<BASE-BRANCH>` is read from project-context. This is the story-branch-vs-base source that the later playbook-integration story (NA-55) reuses. Dual diff-source selection (story-branch-vs-develop **vs** merged-commit) is **out of scope** here — see Out of Scope.
3. **Affected-row resolution.** For each **enabled** manifest row, look up its registry row in `refs/doc-types.md` and keep only rows whose `trigger` contains `sync` (v1: the seven `auto` reference/meta rows + the `how-to` row). A row is **affected** when the changed-file set intersects its `source-of-truth` inputs, per the source-map below. `llms-txt` is affected on **every** run (AC4).
4. **Deterministic regen (auto rows, no gate — AC3).** For each affected `auto` row, regenerate its reference-doc set into the row's `target-path` deterministically from its `source-of-truth` frontmatter — idempotent (re-running with no source change yields a byte-identical tree). `llms-txt` regenerates every run.
5. **Narrative drafts (draft-for-review rows, gated — AC3).** For each affected `how-to` row, draft a **refresh** of the how-to page(s) whose source intersects the diff, using the `writing-docs` skill. These drafts are **held at the founder-confirm gate** at the command layer — never written un-confirmed.
6. **Founder-confirm gate (command layer).** Present the deterministic regen summary (informational — auto rows are not gated) and each narrative draft (gated). The founder may accept/edit/reject each narrative draft. The gate is **skipped** when there are zero narrative drafts — the run proceeds straight to the write phase.
7. **Write + PR only on change (AC6).** Write the deterministic regen + `llms.txt` + founder-confirmed narrative drafts. If, after writing, `git status --porcelain` on the target paths is **empty** (deterministic output was byte-identical and no narrative draft confirmed) → no commit, no PR, exit cleanly. Otherwise commit and **open or update** the sync PR.

### Source-of-truth map (diff-file → affected doc-type)

`sync` derives "affected" from this map (the registry's `source-of-truth` column made concrete against the SDLC plugin layout; a consumer repo's equivalent globs resolve from its own project structure):

| Doc-type | `generation-mode` | Affected when the diff touches… |
| --- | --- | --- |
| `command-reference` | auto | any `commands/**` file |
| `agent-reference` | auto | any `agents/**` file |
| `skill-reference` | auto | any `**/SKILL.md` |
| `config-reference` | auto | `.claude/project/project-context.md` or a config template under `refs/*-template.md` |
| `hooks-contract` | auto | `.claude/settings*.json` hooks blocks or any hook script referenced therein |
| `error-reference` | auto | any file whose diff hunk changes an "Error Handling" / error table section in a command/agent/ref |
| `llms-txt` | auto | **always** (AC4) |
| `how-to` | draft-for-review | any source file listed in an existing how-to page's `source:` frontmatter (see Narrative "affected" rule) |

### Narrative ("affected how-to") rule — v1 bound

To keep v1 low-hallucination, `sync` only **refreshes existing** how-to pages; it never invents a brand-new how-to page (authoring new narrative is `seed` mode's job, out of scope here). An enabled `how-to` row is affected when the diff intersects the `source:` frontmatter list of an already-published page under the row's `target-path`. Each such page yields one gated refresh draft. A how-to page with no `source:` frontmatter is never auto-refreshed by `sync`.

### llms.txt format (v1 decision)

`sync` regenerates `llms.txt` at the `llms-txt` row's `target-path` (registry default `llms.txt`, repo-root) deterministically on every run: an **index-only** file (no full-content `llms-full.txt` in v1), grouping every **enabled, `public: yes`** manifest row's generated pages by Diátaxis quadrant, each entry a `title — one-line description — relative link` line derived from the generated page's frontmatter. Idempotent, no narrative synthesis. (Resolves the Epic's llms.txt open question for v1 in the narrowest deterministic form; `llms-full.txt` remains a later enhancement.)

## Pipeline Contract — `refs/docs-pipeline.md`

`docs-pipeline.md` is authored to **mirror `refs/adr-pipeline.md`'s shape** (copy the skeleton; do not abstract a shared ref — same "copy the shape, do not generalize" rule `doc-types.md` and the Epic both state). It is referenced by both `commands/docs.md` and `agents/knowledge-engineer.md` so the contract lives in exactly one place. Required sections:

1. **Purpose + ownership note** — single-source pipeline for `/sdlc:docs`; all writes land under paths resolved from the consumer repo's manifest/project-context; a plugin-authoring write (touching `plugins/**` in this SDLC repo) stays within the `ai-enablement-engineer` write-scope.
2. **Two-phase dispatch across the confirmation boundary** — a dispatched subagent cannot pause for interactive input, so the founder-confirm gate lives at the command layer between two `knowledge-engineer` dispatches, identical in spirit to `adr-pipeline.md` §2 and `/sdlc:analyze`'s scan-then-apply split:
   - **Phase 1 (compute & draft, writes nothing):** resolve manifest → affected rows; produce the deterministic regen content for auto rows + `llms.txt`; draft narrative (how-to) refreshes via `writing-docs`; return the regen summary, the deterministic content, and the narrative drafts to the command layer.
   - **Founder-confirm gate (command layer):** present the informational auto-regen summary and each gated narrative draft; founder accepts/edits/rejects narrative drafts. Skipped when zero narrative drafts.
   - **Phase 2 (write confirmed, fresh dispatch):** the command hands phase 2 the deterministic content **and** the founder-confirmed narrative drafts **verbatim** (inline or via session temp-dir files by path, per `scripts/tmp-dir.sh` — the same payload-contract discipline as `adr-pipeline.md` §2). Write everything, then commit/push/open-or-update the PR — but only if content changed (AC6).
3. **Deterministic regen algorithm** — the reference-doc and `llms.txt` regeneration is a **prose algorithm executed inline, not a committed script** (matches the ADR index-regen algorithm in `adr-pipeline.md` §10 and the plugin's "instructions not code" style). Idempotent: re-running with no source change → byte-identical output.
4. **Voice/format resolution** — narrative drafting resolves voice + output format via `writing-docs`'s chain: `.claude/project/docs-manifest.md` "Voice & format" section → `.claude/project/project-context.md` → stated default. Never hardcoded.
5. **No-op / change-gate semantics** — manifest-absent silent no-op (AC5); commit/PR only on actual content change (AC6); `llms.txt` regenerated every run but committed only if changed (AC4 + AC6).
6. **Branch / PR naming + control flow** — see below.

## Agent Extension — `knowledge-engineer.md`

The existing `knowledge-engineer` agent (owner of the knowledge lifecycle) gains the docs-sync dispatch role. Modifications:

- **Frontmatter `description`** — extend to state it also runs the `/sdlc:docs` sync pipeline (currently reads "Triggered manually via /sdlc:adr").
- **Role & scope** — add: in a docs-sync dispatch it regenerates frontmatter-driven reference docs + `llms.txt` deterministically and drafts gated how-to refreshes, writing under the manifest's resolved `target-path`s (and, when authoring plugin changes in this SDLC repo, only within the `ai-enablement-engineer`-owned surface — the existing Active-guard scope note already covers this).
- **Pipeline pointer** — point at `refs/docs-pipeline.md` as the single source for the docs-sync procedure (as it already points at `refs/adr-pipeline.md` for ADRs). Do not re-inline.
- **Skill loading** — `writing-docs` is already declared as a load-before-drafting skill in the agent (NA-58 wired this); the docs-sync narrative path is exactly the trigger it describes. No new tool grants are required — `sync` needs `Read`/`Write`/`Edit`/`Bash`/`Skill`, all already in the agent's `tools:` allowlist. The claude-mem MCP tools are not used by `sync`.
- **`Skills loaded:` return line** — sync dispatches list `writing-docs` (when a narrative draft was produced), `conventional-commit`, `gh-cli`, `verification-before-completion`, or `none`.

## Branch / PR / Control Flow

| Item | Value |
| --- | --- |
| Branch | `docs/sync-<STORY-KEY>` (off `<BASE-BRANCH>` from project-context — never assume `main`) |
| Commit | `docs(docs): sync <STORY-KEY> reference docs` (via `conventional-commit`) |
| PR title | `docs(docs): sync <STORY-KEY>` |
| PR behaviour | **Open or update** (AC6): before creating, check for an existing PR on `docs/sync-<STORY-KEY>` (`gh pr list --head …`); if present, push the new commit to update it rather than duplicating |
| Control-flow tail | Mirror `commands/adr.md`: after the PR is raised, drive the review loop to convergence via `/loop /sdlc:loop <PR_URL>` (falling back to `ScheduleWakeup` if the harness cannot nest `/loop`); if a terminal STOP hit before any PR (manifest absent, or nothing changed), release directly via `scripts/session-complete.sh` |

## Permissions Detail

Not a multi-role RBAC surface. The single actor is the **founder** running `/sdlc:docs`. The one authority boundary is the **founder-confirm gate**: auto (deterministic) doc types are written without confirmation; narrative (how-to) drafts are written only after explicit founder confirmation at the command layer. A dispatched subagent never runs the gate itself (it cannot pause for input) — the command layer owns it, exactly as `commands/adr.md` does.

## Error Handling

| Scenario | Behaviour |
| --- | --- |
| `.claude/project/docs-manifest.md` absent | Silent no-op — no branch, no dispatch, no PR, no error, no output (AC5) |
| Empty `$ARGUMENTS` / unrecognised first token | STOP with the usage message |
| Future mode token (`release`/`seed`/`audit`/`distill`) | Print "mode not yet implemented (see NA-50)" and exit cleanly — not an error |
| `refs/doc-types.md` unreadable/malformed | Surface the failure and STOP — never regenerate from a partial registry |
| Manifest present but no enabled `sync`-triggered row affected by the diff, and `llms.txt` unchanged | Clean no-op — no commit, no PR (AC6) |
| Deterministic regen produced byte-identical output and no narrative draft confirmed | No commit, no PR (AC6) — write phase detected an empty `git status --porcelain` |
| Founder rejects all narrative drafts but deterministic content changed | Still commit + PR the deterministic regen (AC6 — content changed) |
| `git diff` yields no changed files (empty story diff) | Only `llms.txt` is (re)generated; commit/PR only if it changed |
| `gh` / `raise-pr.sh` failure | STOP and surface — the write is on a branch and reviewable; never leave a raised-but-broken state silently |

## Out of Scope

- **release / seed / audit / distill modes** — own stories (NA-53 … NA-56). `docs.md` ships them as clean "not yet implemented" stubs, not errors.
- **Dual diff-source selection** (story-branch-vs-develop **vs** merged-commit) — added by the playbook-integration story (NA-55). v1 uses the single story-branch-vs-base source.
- **Authoring brand-new narrative pages** (new how-to / tutorial / concept) — that is `seed` mode. `sync` only refreshes **existing** how-to pages whose `source:` intersects the diff.
- **`llms-full.txt`** (full-content variant) — v1 ships index-only `llms.txt`.
- **Building or hosting a docs site** — Mintlify is this repo's choice; the plugin stays stack-agnostic via manifest `target-path`s (Epic out-of-scope).
- **Absorbing `/sdlc:adr` into `/sdlc:docs`** — NA-56. The standalone `/sdlc:adr` command and `refs/adr-pipeline.md` are untouched by this story.
- **Auto-triggering on CI / tag events** — later enhancement (Epic out-of-scope).
