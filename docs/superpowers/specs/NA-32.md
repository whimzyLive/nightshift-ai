## Overview

Build the above-the-fold conversion path of the nightshift home page — Hero (value line, install snippets, animated terminal, GitHub star CTA), the phrase-ticker marquee, the count-up Proof bar, and the Problem framing — as a pixel-faithful recreation of the approved design handoff inside `apps/marketing`. All copy and numbers in these sections are factual/reference content and therefore stay **static in code** (per the NA-29 CMS-vs-static split); nothing here is CMS-driven.

## Scope boundary (this story vs the rest of Home)

In scope: design sections **1 (Hero)**, the **phrase-ticker marquee** that divides Hero from Proof, **2 (Proof bar)**, and **3 (Problem)**. Everything from section 4 (How it works) onward, and all site chrome (nav/footer/starfield/cursor-glow), is out of scope — see Out of Scope.

## Data Model

**None.** This story adds no database entities, no migrations, and no Payload collections/globals. Hero copy, install commands, proof-bar numbers, the terminal script, and the three problem cards are all factual/reference content and stay hard-coded in the React tree (NA-29 constraint: "tooling facts, agent rosters, commands and install snippets stay static in code"). If any of this were CMS-managed it would contradict the epic — it is deliberately not.

## API Surface

**None.** No new HTTP endpoints, route handlers, or server actions. The page is statically rendered marketing content. The only outbound interactions are:

- **Clipboard write** — browser `navigator.clipboard.writeText` (client-side, no network).
- **GitHub star link** — a plain outbound `<a target="_blank">` to `https://github.com/whimzyLive/nightshift-ai` (no API call).

Because there are no endpoints, the API permission table is not applicable (see Permissions).

## Component Architecture

Two homes, following the existing split in this repo:

- **`packages/ui` (web-engineer, `@nightshift-ai/ui`)** — reusable, page-agnostic primitives. Export each from `packages/ui/src/index.ts` with its `*Props` type, mirroring `Badge`/`CtaButton`/`NightSky`.
- **`apps/marketing/src/components/home/`** (new dir) — home-specific section compositions that assemble the primitives with the approved static copy.

### New primitives — `packages/ui/src/lib/nightshift/`

| File                  | Component        | Client?        | Responsibility                                                                                           |
| --------------------- | ---------------- | -------------- | -------------------------------------------------------------------------------------------------------- |
| `install-snippet.tsx` | `InstallSnippet` | `'use client'` | Mono command row with a `$` prompt and a copy-to-clipboard affordance + visual confirmation.             |
| `terminal.tsx`        | `Terminal`       | `'use client'` | Faux terminal chrome that plays a scripted line-by-line run on loop, with magnetic tilt + ambient drift. |
| `count-up.tsx`        | `CountUp`        | `'use client'` | Animates an integer from 0 → target once, the first time it scrolls into view.                           |

Rationale for `'use client'`: all three need browser-only APIs (`navigator.clipboard`, `IntersectionObserver`, `requestAnimationFrame`, pointer events, `matchMedia`). Everything else on the page stays a server component.

#### `InstallSnippet`

```typescript
export interface InstallSnippetProps {
  /** The exact command text — copied verbatim, never re-derived. */
  command: string;
  /** Leading prompt glyph shown before the command. Default: '$'. */
  prompt?: string;
  className?: string;
}
```

Behaviour:

- Renders `prompt` + `command` in `--font-mono`, sharp-cornered (radius 0), on `--surface-terminal`.
- On click of the row (or its copy button), copy the **exact** `command` string via `navigator.clipboard.writeText(command)`.
- On success show a transient confirmation (icon/label swap to a "copied" state) for ~1.5s, then revert.
- Fully keyboard-operable: the copy control is a real `<button>` (Enter/Space activate) with an accessible label (e.g. `aria-label={`Copy: ${command}`}`).
- Clipboard-blocked / unsupported fallback (AC2): if `navigator.clipboard` is undefined or `writeText` rejects, do **not** throw — the command text stays visible and user-selectable so it can be copied manually; optionally show a "select to copy" hint. No error dialog.

#### `Terminal`

```typescript
export type TerminalLineTone = 'default' | 'muted' | 'accent' | 'success';

export interface TerminalLine {
  /** Optional prompt glyph (e.g. '$') rendered before the text. */
  prompt?: string;
  /** Optional agent label rendered as a colored tag before the text. */
  agent?: string;
  /** Line body text. */
  text: string;
  /** Semantic color tone. Default: 'default'. */
  tone?: TerminalLineTone;
  /** Indent depth (each level = one step of left padding). Default: 0. */
  indent?: number;
}

export interface TerminalProps {
  /** Chrome title-bar caption. */
  title: string;
  /** The scripted lines, revealed in order. */
  lines: readonly TerminalLine[];
  /** Min height of the body in px (reserve space to avoid layout shift). */
  minHeight?: number;
  className?: string;
}
```

Behaviour (AC4):

- Renders a title bar (`title`, e.g. `zsh — acme-api · claude code`) plus a body that reveals `lines` **one at a time** on a timer (`--dur-terminal-line`, 520ms cadence), then **loops** back to the start.
- A blinking caret is shown on the final prompt line.
- **Magnetic tilt on hover:** while the pointer is over the wrapper, apply a subtle 3D tilt tracking the pointer (`transform-style: preserve-3d`, matching the design's `#ns-term-wrap`). On pointer leave, ease back to rest.
- **Ambient drift otherwise:** when not hovered, a slow idle float.
- **Reduced motion:** when `matchMedia('(prefers-reduced-motion: reduce)').matches`, render **all** lines immediately (no line-by-line reveal, no loop), and attach **no** pointer-tilt / drift listeners — mirror the self-guarding pattern in `night-sky.tsx` / `cursor-glow.tsx` (check `matchMedia` in an effect and bail before wiring listeners), not just rely on the CSS guard.
- The scripted content is passed in by the caller (see Hero) — the primitive is content-agnostic.

#### `CountUp`

```typescript
export interface CountUpProps {
  /** Target integer to count to. */
  value: number;
  /** Rendered before the number (e.g. ''). Default: ''. */
  prefix?: string;
  /** Rendered after the number (e.g. ' seconds'). Default: ''. */
  suffix?: string;
  /** Animation duration in ms. Default: ~1200. */
  durationMs?: number;
  className?: string;
}
```

Behaviour (AC5):

- Uses one `IntersectionObserver` to detect first entry into the viewport, then animates `0 → value` once via `requestAnimationFrame` and disconnects (never re-runs).
- Reduced motion: render the final `value` immediately, skip the observer + rAF entirely.
- Renders the number inline so the surrounding static label text ("specialized agents", etc.) stays in the parent.

## Web UI

**Page:** `apps/marketing/src/app/(frontend)/page.tsx` (existing — currently a placeholder heading). Replace its body with an ordered composition:

```
<Hero />
<PhraseMarquee />
<ProofBar />
<ProblemSection />
```

The page remains a server component and stays inside the existing `(frontend)/layout.tsx` shell (NightSky/CursorGlow/NavBar/Footer/`<main>` container are already provided by NA-30 — do not re-add them). Note: the current `<main>` applies `max-w-[var(--container-max)] px-7`; the Proof bar and section backgrounds in the design are **full-bleed** (edge-to-edge tinted bands). Implementer decision required — see Open Questions — default is to let full-bleed bands break out of the container while keeping inner content centered at `--container-max`.

**Layout specifics (from the handoff):**

- Hero is a two-column grid `1fr 1.12fr`, `gap:48px`, `align-items:center`; collapses to a single column on narrow viewports (stack terminal under the copy). Match the responsive behaviour shown in screenshots 01/02-home.png.
- Proof bar: `--surface-card` band with top/bottom `--border-default` hairlines; inner content centered, wraps on narrow widths.
- Problem: centered header/body max ~700–900px; three-column card grid (`repeat(3,1fr)`, `gap:16px`) collapsing to one column on mobile.
- Sharp edges everywhere (radius 0) except circular dots — the `--radius-*` tokens are already 0 in `global.css`.

**Design tokens / styling:** Tailwind v4 utilities + `var(--token)` inline styles, consistent with `Badge`/`CtaButton`/`NightSky`. No CSS Modules/SCSS. All colors, type sizes, spacing come from the existing `@theme` tokens in `apps/marketing/src/app/global.css`.

**`global.css` additions (keyframes only — no new tokens):** add the two keyframes the design uses that are not yet present: `ns-marquee` (0 → -50% translateX, paired with `--dur-marquee`) and `ns-blink` (terminal caret). Both must be covered by the existing `@media (prefers-reduced-motion: reduce)` guard. Do not touch the `@theme` block.

**Data hooks:** none — no server data fetching (static content).

**State:** none global — the only state is component-local (`InstallSnippet` copied flag, `Terminal` current line index, `CountUp` progress).

**Route permissions:** `/` (the frontend home) is a public marketing route with no auth — unchanged from NA-30.

## Permissions

**Not applicable.** This story introduces no authenticated endpoints, roles, or gated data. The home route is public and unauthenticated. There is no per-role permission matrix because there is no protected surface — every visitor sees identical content.

## Motion & Accessibility

| Concern        | Requirement                                                                                                                                                                                                                                                                                                               |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Reduced motion | Terminal renders full/static, no loop; ProofBar shows final numbers; marquee frozen; terminal tilt/drift not wired. Each JS-driven effect self-checks `matchMedia`, not just the CSS guard.                                                                                                                               |
| Keyboard       | Install-snippet copy is a real `<button>`, focusable, Enter/Space activate; GitHub CTA is a focusable link. Focus-visible uses the global `--glow-focus` ring.                                                                                                                                                            |
| Screen readers | Marquee ticker is `aria-hidden="true"` (decorative). Copy buttons have an `aria-label` including the command. Terminal is decorative/illustrative — do not force SR to read the scripted animation loop; expose a static accessible summary or mark the animated body `aria-hidden` with the key claim available as text. |
| Hydration      | `Terminal`/`CountUp` must render a deterministic server frame (line 1 / value shown or 0) so hydration matches, following the deterministic pattern used in `night-sky.tsx`.                                                                                                                                              |

## Error Handling

| Scenario                           | Behaviour                                                                                                                                                         |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `navigator.clipboard` unavailable  | No throw; command stays visible/selectable for manual copy; optional "select to copy" hint (AC2).                                                                 |
| `clipboard.writeText` rejects      | Same graceful fallback as above; never surface a blocking error.                                                                                                  |
| `prefers-reduced-motion: reduce`   | All animation suppressed as above; content fully rendered and functional.                                                                                         |
| `IntersectionObserver` unsupported | `CountUp` renders the final value immediately (degrade to static).                                                                                                |
| JS disabled / pre-hydration        | Hero copy, install commands (as selectable text), proof numbers (final values), and problem cards are all present in server HTML; only the animations are absent. |

## Out of Scope

- Site-wide chrome — nav, footer, starfield (`NightSky`), cursor-glow — delivered by NA-30 (chrome story); this story consumes the existing layout shell, it does not build it.
- Home sections 4 onward — How it works (`Pipeline`), Day/night workflow, The team, Why different, Control bridge, FAQ preview, final CTA — built in the following home stories.
- The `Pipeline`, org-chart/tree, and triage-playground interactive widgets from later sections.
- Any Payload CMS collection/global, migration, or seed change — these sections are static in code by design (NA-29).
- Analytics / instrumentation of install-copy and star click-through — explicitly out for v1 (NA-29 open questions).
- Scroll-reveal / `.ns-stagger` staggered entrance polish — not required by the ACs (AC5 requires the cards to render, not to animate in); if desired it is a later polish story, to avoid pulling the global reveal system into this one.
- Porting the prototype `support.js` runtime — only the approved content/copy is reused, not the engine (NA-29).

## Open Questions

- [ ] Full-bleed bands vs the `max-w-[var(--container-max)]` `<main>` container: the Proof bar and section background bands are edge-to-edge in the design, but the layout shell wraps children in a centered, padded `<main>`. — **Suggested default:** let full-bleed bands break out to viewport width (negative-margin / `w-screen` escape) while keeping inner content centered at `--container-max`; do not widen or remove the shared `<main>` container (other stories depend on it).
- [ ] Where the `ns-marquee` / `ns-blink` keyframes live: `global.css` vs the ui package. — **Suggested default:** add them to `apps/marketing/src/app/global.css` next to the existing `ns-drift`/`ns-twinkle` keyframes, so the reduced-motion guard already covers them.
- [ ] Visual-parity comparison tooling (pixel-diff vs side-by-side visual review). — **Suggested default:** side-by-side visual review against `screenshots/01–03-home.png` in the QA phase (the existing `apps/marketing-e2e` Playwright project captures the built screenshots); a strict pixel-diff threshold is not required and font/AA rendering differences should not fail the gate.
