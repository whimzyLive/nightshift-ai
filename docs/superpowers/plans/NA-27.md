# Cut per-dispatch worktree and agent cold-start overhead — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/NA-27.md](../specs/NA-27.md)
**Story:** NA-27 (parent epic NA-24 "SDLC Improvements")
**Date:** 2026-07-12
**Agents required:** `ai-enablement-engineer` (only) — pure SDLC-plugin tooling, dependency-free domain

**Goal:** Replace the harness's per-dispatch `isolation: "worktree"` in the impl phase with one orchestrator-managed per-story worktree, a shared Nx cache, a merged-only worktree GC, and a flag-controlled agent-reuse path (shipped `enabled`).

**Architecture:** Two new bash scripts under `plugins/sdlc/scripts/` (idempotent worktree provision + merged-only GC) wired into `session-complete.sh` and the `SessionEnd` hook; playbook rewrites in `refs/principal-engineer-playbook.md` and `refs/qa-engineer-playbook.md` so the orchestrator owns isolation, threads `WORKTREE` + `NX_CACHE_DIRECTORY` into every dispatch, machine-checks the primary checkout, and branches fix-round dispatch on a reuse flag; prose mirrors in the two commands and two plugin agent docs; capability tokens in `.claude/project/project-context.md`.

**Tech Stack:** Bash (POSIX-ish, `set -uo pipefail` house style matching sibling scripts), `git worktree`, pnpm workspace, Nx 23.0.1 (`NX_CACHE_DIRECTORY` env), Markdown playbooks/commands/agent docs.

## Execution order

Single phase — every file this story touches lives under `plugins/sdlc/**` plus `.claude/project/project-context.md`, all owned by `ai-enablement-engineer` (a dependency-free domain: it consumes no artifact from and produces no artifact for any other domain agent). No database / platform / sync / web / mobile phases apply. Tasks are executed **in order** on the shared `feat/NA-27` branch, one at a time: new scripts first, then hook/session wiring, then playbook edits, then command + agent-doc prose mirrors, then project-context tokens, then the phase verification gate.

## Global Constraints

Copied verbatim from the spec — every task's requirements implicitly include these:

- **Scope: impl phase only.** Only `principal-engineer` + `qa-engineer` domain/fix dispatches (including the `WORK_KIND=defect` variant) change. The spec / plan / refine phases (`solutions-architect`, `tech-lead`, `scrum-master`, `product-manager`) stay on the **main checkout, unchanged** — no worktree, no cache override, no reuse flag added there.
- **Worktree path (deterministic):** `.claude/worktrees/sdlc-<STORY-KEY>` — distinct from the harness's `agent-<hash>` dirs; already gitignored via `.claude/worktrees`.
- **Shared Nx cache path:** `NX_CACHE_DIRECTORY=<repo-root>/.nx/cache` (absolute; the **primary checkout's** cache; already gitignored). Never mutate committed `nx.json`.
- **GC safety gate (the ONLY safe removal condition):** remove a worktree iff BOTH `WB != <BASE-BRANCH>` (its branch is not the base itself) AND `git merge-base --is-ancestor "$WH" origin/main` succeeds (fully merged into `origin/main`). Base is `origin/main`, always defined even with no story-key arg.
- **Base branch is `main`** (NOT develop). All merge-gate resolution uses `origin/main`.
- **GC never fails a session:** `worktree-gc.sh` always `exit 0` (mirrors `cleanup-tmp.sh`).
- **node_modules provisioning:** `pnpm install --prefer-offline` (optionally `--frozen-lockfile`) once per worktree — never a root-only symlink (pnpm workspace needs per-package link farms).
- **Reuse flag ships enabled:** `SDLC agent reuse = enabled` in project-context Tooling (owner Open-Question-4 decision, 2026-07-11). `disabled` is the documented off-switch. This story only _consumes_ the flag; it does NOT fix the frontmatter re-injection bug (#76337 = NA-23).
- **Primary checkout NEVER checks out the story branch** — the branch is created _inside_ the worktree (`git worktree add -b`). All orchestrator git ops on the story branch run via `git -C "$WORKTREE"`.
- **nx-generated mirror boundary — DO NOT HAND-EDIT:** `agents/`, `.codex/`, `.opencode/`, `.gemini/`, `opencode.json` at the repo root are machine-maintained mirrors regenerated by the nx generator. This story touches **none** of them. (Verified: the SDLC plugin agent docs are NOT surfaced in those roots — root `agents/` holds only `ci-monitor-subagent.md` — so editing `plugins/sdlc/agents/*.md` requires no regeneration.)

### Task 1: New script `worktree-setup.sh` (idempotent per-story provision + branch ownership)

**Files:**

- Create: `plugins/sdlc/scripts/worktree-setup.sh`

**Interfaces:**

- Invocation: `bash worktree-setup.sh <STORY-KEY> <BRANCH> <BASE-BRANCH>` — e.g. `worktree-setup.sh NA-27 feat/NA-27 main`.
- Produces (stdout, exactly these two capture lines for the orchestrator to grep): `WORKTREE=<abs path>` and `NX_CACHE_DIRECTORY=<abs path>`.
- Consumes: nothing from other tasks.

- [ ] **Step 1: `[ai-enablement-engineer]` Create the script with the house-style header and arg guard.** `#!/usr/bin/env bash`, `set -uo pipefail` (match `session-complete.sh` / `cleanup-tmp.sh`). Require exactly 3 args (`STORY_KEY`, `BRANCH`, `BASE_BRANCH`); on missing args print usage to stderr and `exit 2`. Resolve the primary repo root once: `PRIMARY_ROOT="$(git rev-parse --show-toplevel)"`.

- [ ] **Step 2: `[ai-enablement-engineer]` Compute deterministic paths.** `WT="$PRIMARY_ROOT/.claude/worktrees/sdlc-$STORY_KEY"` and `NX_CACHE="$PRIMARY_ROOT/.nx/cache"`. Do NOT create `.nx/cache` here (Nx creates it on first task run); it only needs to be an absolute path under the primary root.

- [ ] **Step 3: `[ai-enablement-engineer]` Implement the three idempotent resolution cases (spec §1, invariant 1).** In priority order:
  1. **`WT` already a registered worktree** (`git worktree list --porcelain` contains `worktree $WT`) → **reuse**: `git -C "$WT" fetch origin "$BRANCH"` then fast-forward to the branch head (`git -C "$WT" merge --ff-only "origin/$BRANCH"` — no-op if already current). A non-fast-forwardable state is a hard failure (Step 6).
  2. **Branch exists** (local `git show-ref --verify --quiet "refs/heads/$BRANCH"` OR remote `git ls-remote --exit-code --heads origin "$BRANCH"`) **but no worktree** (re-provision-after-teardown) → `git worktree add "$WT" "$BRANCH"` (track the existing branch). First `git fetch origin "$BRANCH"` so the local ref exists.
  3. **Neither exists** (first run) → `git fetch origin "$BASE_BRANCH"` then `git worktree add -b "$BRANCH" "$WT" "origin/$BASE_BRANCH"` (creates the branch **inside the worktree** from the base; the primary checkout is untouched — invariant 1).

- [ ] **Step 4: `[ai-enablement-engineer]` Provision `node_modules` once per worktree (spec Open Q1, decided).** After the worktree exists, if `[ ! -d "$WT/node_modules" ]` run `pnpm install --prefer-offline --frozen-lockfile` with `cwd="$WT"` (via `git -C` is not applicable — use a subshell `( cd "$WT" && pnpm install --prefer-offline --frozen-lockfile )`). pnpm's content-addressed store hardlinks across worktrees, so this is cheap. A genuine pnpm failure (non-zero) → hard failure (Step 6). Do NOT symlink the primary `node_modules` (rejected in spec — breaks per-package link farms).

- [ ] **Step 5: `[ai-enablement-engineer]` Emit the two capture lines.** On success print exactly:

  ```
  WORKTREE=<abs $WT>
  NX_CACHE_DIRECTORY=<abs $NX_CACHE>
  ```

  These are the ONLY two lines the orchestrator parses — keep them on their own lines, `KEY=value`, absolute paths.

- [ ] **Step 6: `[ai-enablement-engineer]` Hard-failure contract (spec Error Handling).** Any failure to create/reuse the worktree, or an unusable env after `pnpm install`, → write a clear message to stderr and `exit 1`. The orchestrator STOPs the impl phase on non-zero — the script must NEVER silently fall back to the primary checkout for a writing agent.

- [ ] **Step 7: `[ai-enablement-engineer]` Self-review the script.** Run `bash -n plugins/sdlc/scripts/worktree-setup.sh` (syntax) and a manual shellcheck-style pass (quote every expansion, `[ ... ]` tests, no unguarded `$1`). There is no repo lint target for shell — this manual pass is the gate. Make it executable: `chmod +x plugins/sdlc/scripts/worktree-setup.sh`.

- [ ] **Step 8: `[ai-enablement-engineer]` Commit.**
  ```bash
  git add plugins/sdlc/scripts/worktree-setup.sh
  git commit -m "feat(sdlc): add idempotent per-story worktree-setup.sh"
  ```

### Task 2: New script `worktree-gc.sh` (merged-into-origin/main-only reclamation)

**Files:**

- Create: `plugins/sdlc/scripts/worktree-gc.sh`

**Interfaces:**

- Invocation: `bash worktree-gc.sh [STORY-KEY]` — the arg is optional; base is always `origin/main` regardless (SessionEnd calls it with no arg).
- Consumes: nothing. Produces: nothing captured — side effect only (removes merged worktrees). Always `exit 0`.

- [ ] **Step 1: `[ai-enablement-engineer]` Create the script with the always-safe contract.** `#!/usr/bin/env bash`, `set -uo pipefail`. Accept an optional `STORY_KEY="${1:-}"` (the story-key arg only narrows logging/enumeration; it does NOT gate removal). `PRIMARY_ROOT="$(git rev-parse --show-toplevel)"`. Wrap the whole body so any error path still reaches `exit 0` (spec: GC must never fail a session — mirror `cleanup-tmp.sh`).

- [ ] **Step 2: `[ai-enablement-engineer]` Resolve the merge base once, always defined (spec §2, Finding 6).** `BASE_BRANCH=main` (project-context Base branch); `BASE="origin/$BASE_BRANCH"`. Refresh first: `git fetch origin "$BASE_BRANCH" 2>/dev/null || true`. Base is defined even with no `STORY_KEY` arg — this is the SessionEnd case.

- [ ] **Step 3: `[ai-enablement-engineer]` Enumerate candidate worktrees (spec Open Q2, decided: both scopes).** From `git worktree list --porcelain`, collect worktrees whose path basename matches `sdlc-*` (this story's managed worktrees) OR `agent-*` (stray harness leftovers under `.claude/worktrees/`). Both are gated on the same merge check below, so an in-flight worktree of either kind is never removed.

- [ ] **Step 4: `[ai-enablement-engineer]` Apply the ONLY safe removal gate to each candidate (spec Global Constraints, Findings 2 & 6).** For each worktree read its branch `WB` (`git -C "$WT" rev-parse --abbrev-ref HEAD`) and head `WH` (`git -C "$WT" rev-parse HEAD`). Remove iff BOTH hold:
  - (a) `[ "$WB" != "$BASE_BRANCH" ]` — never the base branch itself; AND
  - (b) `git merge-base --is-ancestor "$WH" "$BASE"` succeeds — its work is fully merged into `origin/main`.
    A detached-HEAD or unreadable worktree fails gate (a)/(b) safely → skip.

- [ ] **Step 5: `[ai-enablement-engineer]` Remove + delete branch on a passed gate.** `git worktree remove --force "$WT"` then delete the now-merged local branch `git branch -D "$WB" 2>/dev/null || true`. Each removal wrapped so one failure (locked worktree, already-removed dir) logs and continues — never aborts the loop.

- [ ] **Step 6: `[ai-enablement-engineer]` Prune stale admin entries and exit 0.** After the loop: `git worktree prune 2>/dev/null || true`. Final line `exit 0` unconditionally.

- [ ] **Step 7: `[ai-enablement-engineer]` Self-review + executable.** `bash -n plugins/sdlc/scripts/worktree-gc.sh`; manual shellcheck-style pass; assert every path returns 0 (no `exit 1`). `chmod +x plugins/sdlc/scripts/worktree-gc.sh`.

- [ ] **Step 8: `[ai-enablement-engineer]` Commit.**
  ```bash
  git add plugins/sdlc/scripts/worktree-gc.sh
  git commit -m "feat(sdlc): add merged-only worktree-gc.sh reclamation"
  ```
