**Goal:** Build Home sections 4‚Äì7 (How it works, Review by day/ship by night, The team preview, Why builders choose it) inside `apps/marketing`, appended after `ProblemSection`, as a high-fidelity verbatim recreation of the approved design handoff.

**Tech Stack:** Next.js (App Router, `(frontend)` route group), React 19, TypeScript (`.tsx` only), Tailwind CSS v4 + inline design-token CSS vars, `@nightshift-ai/ui` primitives. Nx monorepo, pnpm.

## Global Constraints

- **Single agent:** web-engineer only. No DB / backend / sync / mobile / CMS / Payload work ‚Äî sections 4‚Äì7 are static factual/reference content per the Epic NA-29 constraint.
- **Verbatim copy:** every heading, body paragraph, table row, card text, timeline label, and tree row is copied verbatim from the design handoff `docs/design/marketing-site-handoff/designs/nightshift Landing.dc.html`. Do NOT re-author. Add a source-line comment per section (`// Verbatim from the design handoff (nightshift Landing.dc.html Lxxx-Lyyy).`), following the `problem-section.tsx` convention.
- **UI layer rule (CLAUDE.md):** all components `.tsx`; Tailwind utilities + inline design-token CSS vars (`var(--...)`); NO CSS Modules / SCSS / `.jsx` / `.js` / class components.
- **Reused primitives (already exported ‚Äî do NOT rebuild):** `Card`, `Eyebrow`, `InstallSnippet` from `@nightshift-ai/ui`. `InstallSnippet` props: `command: string`, `prompt?: string` (default `'$'`), `className?`. The `TeamPreview` terminal-tree is bespoke plain markup, NOT the `Terminal` primitive.
- **Full-bleed band pattern:** sections needing a viewport-wide `bg-void` + `border-top` band use the established `problem-section.tsx` pattern ‚Äî `className="relative left-1/2 right-1/2 -mx-[50vw] w-screen ..."` with an inner max-width container.
- **Reduced motion (AC5):** every hover/twinkle effect must degrade to a static, legible layout under `prefers-reduced-motion: reduce`. Gate the tree twinkle behind `motion-safe`. Default (no-hover) state of every section must be fully legible on its own.
- **Verification per phase:** `pnpm nx run-many -t test` (repo test command). Lint + typecheck via the affected marketing/ui projects.
