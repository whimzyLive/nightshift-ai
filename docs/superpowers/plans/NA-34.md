# You decide how it gets built (control section) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL — use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/NA-34.md](../specs/NA-34.md)
**Story:** NA-34 — Build home you-decide-how-its-built control section (Epic NA-29, marketing site)
**Date:** 2026-07-12
**Agents required:** `web-engineer` (sole domain — apps/marketing + packages/ui)

**Goal:** Ship Home section 8 ("You decide how it gets built") — three connected interactives (triage router, route-aware gate strip, side-by-side comparison terminals) driven by ONE client-side state machine.

**Architecture:** A single `'use client'` React island (`ControlSection`) owns the shared state machine; the triage router, gate strip, and nightshift terminal all derive their view purely from that state so they can never desync (AC4). The one-shot AI terminal runs an independent scripted `raceStep` timer. All content is inline design-verbatim constants — no server data, no network, no DB, no Payload.

**Tech Stack:** Next.js App Router (React 19) · Tailwind v4 CSS-first · `@nightshift-ai/ui` kit · all `.tsx`, Tailwind utilities + inline CSS-variable tokens (no CSS modules / SCSS / `.jsx`).

## Global Constraints

- All UI is `.tsx` — never `.jsx`, `.js`, or class components. Styling is Tailwind utilities + inline CSS-variable tokens. No CSS modules, no SCSS, no hand-rolled stylesheets.
- Sharp edges everywhere (`border-radius: 0`) on cards, buttons, terminal frames, route lanes, gate labels; only gate circles stay round (`border-radius: 50%`).
- All colors via existing CSS-variable tokens (`--bg-void`, `--surface-card`, `--surface-terminal`, `--border-*`, `--moon-*`, `--text-*`, `--terra-*`, `--accent`, `--cyan-400`/`--green-400`/`--amber-400`/`--red-400`, `--link`, `--font-mono`, `--ease-out`). Never hardcode hexes.
- All literal copy (lane titles/notes/paths, `triageMsg` variants, `approvalModeHint` variants, `CTRL_MAP` strings, one-shot failure lines, right-terminal triage/artifact/PR lines) is **approved copy — reproduce verbatim** from the design `renderVals` / `CTRL_MAP` / `LINES`. Do not rewrite.
- Design source of truth: `docs/design/marketing-site-handoff/designs/nightshift Landing.dc.html` section 8 (`data-screen-label="Control bridge"`, `class Component` L620–1340, `renderVals` L1083–1340) + `docs/design/marketing-site-handoff/README.md` (§ Home section 8, § State Management, § Design Tokens).
- State field names/semantics are fixed by the design and MUST be preserved verbatim (see Task 3 type).
- Every derived view value is **recomputed each render, never stored** — this is the AC4 no-desync guarantee.
- Every timer-driven effect self-checks `prefers-reduced-motion` and shows end-states immediately when reduced (AC5). All `setTimeout`/`setInterval` handles are cleared on unmount and before every restart.

### Task 3 — Build the `ControlSection` state machine + shell (root client island)

**Files:**

- Create: `apps/marketing/src/components/home/control-section.tsx` (`'use client'`, exports `ControlSection`).

State container (`useReducer` recommended for the discrete gate transitions + config-change resets; `useState` acceptable). Preserve these field names/semantics verbatim (design `class Component` L621, `renderVals` L1083–1340):

```typescript
type TicketType = 'story' | 'bug';
type ApprovalMode = 'assisted' | 'auto' | 'full-auto';
type GateStatus = 'working' | 'awaiting';

interface ControlState {
  ticketType: TicketType; // default 'story'
  storyPts: number; // 1–13, default 8 (only meaningful when ticketType === 'story')
  thresh: number; // lightweight_threshold, clamped 1–8, default 3
  approvalMode: ApprovalMode; // default 'assisted'
  gI: number; // index of current gate within active route
  gS: GateStatus; // current gate phase: 'working' then 'awaiting'
  gDone: boolean; // whole route completed
  raceStep: number; // 0–12, one-shot terminal scripted progression
}
```

- [ ] `[web-engineer]` Scaffold `ControlSection` with `'use client'`, the section shell: full-bleed band `background: var(--bg-void)`, top hairline `border-color: var(--border-default)`, `padding: 72px 28px`, centered header column `max-width: 760px`, comparison grid `max-width: 1120px`. Render `<Eyebrow>// 06 · control</Eyebrow>` (from `@nightshift-ai/ui`), the `You decide how it gets built` H2, and the intro paragraph — copy verbatim from the design.
- [ ] `[web-engineer]` Implement `_route(ticketType, storyPts, thresh)` pure fn: `bug` → `['implement','review']`; else `storyPts <= thresh` → `['refine','implement','review']`; else → `['refine','spec','plan','implement','review']`. (Gate counts 5 / 3 / 2 → AC2.)
- [ ] `[web-engineer]` Implement the gate lifecycle: `_startGate(0)` runs on mount AND on every config change (`setStoryPts`, `incThreshold`/`decThreshold` clamped 1..8, `setTypeStory`/`setTypeBug`, `cycleMode`) — it `clearTimeout`s the pending gate timer first, then restarts the walk from gate 0 (AC4 re-sync). Per gate: `gS:'working'`, after ~1700ms (~400ms under reduced motion) → `gS:'awaiting'`; if `_gateAuto(i)` for the current mode, schedule `_advanceGate()` ~700ms later. `_advanceGate` → next gate, or `gDone:true` at end; under `full-auto` re-loop via `_startGate(0)` after ~3200ms.
- [ ] `[web-engineer]` Implement `_gateAuto(i)`: `assisted` → never auto; `auto` → auto every gate EXCEPT `spec` and `review`; `full-auto` → auto every gate and loop. `approveGate` advances only when `gS==='awaiting' && !gDone` (no-op guard otherwise). `resetGates` = `_startGate(0)`.
- [ ] `[web-engineer]` Implement the one-shot `raceStep` timer: independent interval incrementing `raceStep` once/second up to 12, plays the scripted failure sequence ONCE (does not loop). Under reduced motion, jump straight to `raceStep:12` (AC3, AC5).
- [ ] `[web-engineer]` Clear ALL `setTimeout`/`setInterval` handles in effect cleanup on unmount (no state-update-after-unmount) and before every `_startGate(0)` restart (no orphaned timers / double-advance).
