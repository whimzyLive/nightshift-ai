## Goal

Add a real-role `knowledge-engineer` agent plus a two-mode `/sdlc:adr` command (seed + `--distill`) that turn founder-known patterns and the accumulated learnings corpus into curated, indexed ADRs under `docs/adr/` through one shared draft → propose-tags → founder-confirm → write → regenerate-index → commit/PR pipeline. Wire domain agents to read a per-agent ADR index instead of re-deriving conventions or re-appending covered learnings.

## Architecture

The pipeline contract lives in exactly one place — `refs/adr-pipeline.md` — referenced by both the agent and the command. The founder-confirmation gate runs at the **command layer** (a dispatched subagent cannot block for interactive input), so the pipeline is split across a two-dispatch boundary: phase-1 dispatch drafts and returns (writes nothing); the command presents drafts + deletions and waits for confirmation; phase-2 dispatch writes only confirmed items and self-raises the PR. `claude-mem` is added as a plugin dependency and its MCP tools are whitelisted in the agent frontmatter so distill can mine observations.

- **Owner / scope:** every path is under `plugins/sdlc/**` → tag `[ai-enablement-engineer]`. ## Grounding corrections & open-question resolutions (carry verbatim into the named tasks)

1. **[GROUNDING CORRECTION — supersedes spec Open Question "Exact claude-mem MCP server slug + tool names"]** The installed `claude-mem` plugin (`thedotmack/claude-mem@13.8.1`) registers its MCP server under the key **`mcp-search`** in `.mcp.json` — **NOT** `claude-mem`. The tool base names the spec assumed are correct and present: `observation_search` and `get_observations` (both verified in `scripts/mcp-server.cjs`). The spec's assumed frontmatter names `mcp__claude-mem__observation_search` / `mcp__claude-mem__get_observations` therefore have the **wrong server slug**. The harness plugin-prefix rendering could not be pinned byte-exact from a static session (the observed live pattern for another plugin is `mcp__plugin_<plugin>_<serverKey>__<tool>`, e.g. `mcp__plugin_context-mode_context-mode__…`). Per the spec's own fallback rule ("fall back to the server-level wildcard only if the tools cannot be pinned"), Task 3 MUST: (a) at implementation time confirm the exact exposed name against the **live** tool list (run `/mcp` or inspect the running session's tool names with claude-mem installed); (b) if pinnable, use the explicit fully-qualified names with the real `mcp-search` slug; (c) if not pinnable, use the **server-level wildcard** for that server. Do **not** ship the literal `mcp__claude-mem__*` strings from the spec — the slug is wrong.
2. **[OQ — knowledge-engineer model tier]** Resolved: **`sonnet`** (matches sibling curation agent `ai-enablement-engineer`; keeps the frequent seed path cheap). Revisit to `opus` only if distill corpus-clustering proves weak on `sonnet` after first real runs.
3. **[OQ — index regeneration: prose vs script]** Resolved: **deterministic prose-algorithm executed inline** by the agent (Read frontmatter → group → sort → write). No committed script — matches the plugin's "instructions not code" style.
4. **[OQ — branch/PR naming for /sdlc:adr]** Resolved: seed → branch `docs/adr-<slug>`, PR title `docs(adr): <decision title>`; distill → branch `docs/adr-distill-<YYYY-MM-DD>`, PR title `docs(adr): distill <n> ADR(s) from learnings corpus`.
5. **[OQ — write-path guard: hard-block vs soft-skip]** Resolved: **soft-skip with a note** (append is skipped, never failed) so a domain agent is never blocked by ADR bookkeeping. Guard is already restricted to `accepted` ADRs — a non-accepted match never skips.
6. **[Active-guard scope]** The `ai-enablement-engineer`-must-be-Active STOP applies **only to plugin-authoring writes** (a dispatch writing under `plugins/**`). A plain ADR run in a consumer repo writes only `docs/adr/**` (+ founder-gated `.claude/memories/**` deletions) and needs no `ai-enablement-engineer` ownership — do not STOP on its absence there.
7. **[shared.md audience-preservation]** A learning promoted from `.claude/memories/agents/shared.md` may be deleted only when the replacing ADR preserves the audience (tagged with every agent the shared learning served, OR routed to `General` which every agent reads). Otherwise the ADR is still written but the `shared.md` entry stays.
8. **[Evidence contract]** Repo-native citations (story keys / PR review threads / commit SHAs) are the **primary** evidence and fully satisfy the gate. Observation IDs are required only for claude-mem-sourced occurrences. Tools absent → **halt** distill (environment gate). Tools present but DB empty → **non-fatal**, continue on repo-native evidence.
9. **[Write-path guard status filter]** The guard suppresses an append **only** against a `status: accepted` ADR. A match against a `superseded` / `rejected` / `proposed` ADR does **not** skip — the learning is still appended. The index line carries status so the guard filters without opening the ADR.
10. **[Seed-mode source-stories]** A seed-mode ADR MAY carry an empty/omitted `source-stories` (there is often no originating Jira story). The `writing-adrs` "source-stories lists the motivating Jira key(s)" checklist item scopes to **distill** mode. `adr-pipeline.md` must state this so a seed ADR with no `source-stories` is not flagged incomplete.

  4. **Distill evidence protocol** — evidence sources: `.claude/memories/agents/*.md` (Read), `.claude/memories/reviews/patterns.md` (Read), PR review threads (`gh`), commit history (`git log`), claude-mem observations (`observation_search` / `get_observations`). **Evidence contract [carry OQ-resolution #8 verbatim]:** every distilled candidate cites evidence; the citation KIND depends on the occurrence's source — claude-mem-sourced occurrences cite observation ID(s) (required only for those); repo-native occurrences cite story key(s) / PR thread(s) / commit SHA(s), no observation ID required; a candidate satisfies the gate when each cited occurrence carries ≥1 valid citation of the appropriate kind; a candidate may mix both. Empty observation DB → a candidate evidenced entirely by repo-native citations **fully satisfies** the contract; `source-stories` is populated from cited story keys regardless.
  5. **claude-mem gates (environment, not data) [carry OQ-resolution #8]:** tools absent → **halt** with a clear message (e.g. `claude-mem tools unavailable — /sdlc:adr --distill requires the claude-mem plugin; install it or use seed mode`); tools present but DB empty → **non-fatal** warning-and-continue on repo-native evidence. State the two are distinct.
  6. **Promotion criteria (distill)** — promote iff ≥1 of: **Recurrence** (recurs across ≥2 cited occurrences, each backed by an appropriate-kind citation), **Cross-agent** (spans >1 sdlc agent's domain), **Durable convention** (stable convention future work must follow, even if seen once, because it's expensive to reverse/re-derive). Candidates failing all three are **not** promoted (reported "below threshold", left in memory).
  7. **`shared.md` audience-preservation rule [carry OQ-resolution #7 verbatim].**
  8. **Deletion-on-promotion** — on write of a confirmed promoted candidate, delete the founder-approved raw learning entries from their source memory files **in the same PR** (git history preserves them); subject to the founder gate and the shared.md rule. Seed mode skips this step.
  9. **Seed-mode `source-stories` scoping [carry OQ-resolution #10 verbatim]** — explicitly state that the `writing-adrs` `source-stories` checklist item applies to distill, not seed, so a seed ADR with no `source-stories` is not flagged incomplete.
  10. **Index Regeneration algorithm (`docs/adr/index.md`)** — deterministic, fully derived from frontmatter, never hand-authored. Grouping: one section per distinct agent named across all `agents:` lists, plus a **`General`** section for every ADR whose `agents` is empty/omitted (no ADR ever dropped); an ADR routed to N agents appears under each. Line content: `- [NNNN. Title](NNNN-slug.md) — <status>`. Ordering: sections alphabetical by agent name with `General` **last**; ADRs within a section ascending by `NNNN`. Idempotent (running twice with no ADR change yields a byte-identical file). Prose-algorithm executed inline (Read frontmatter → group → sort → write) — **no committed script** [OQ-resolution #3].
  11. **Cross-reference:** point to the `analyze-protocol.md` "Exception 2" anchor (Task 5) for the sanctioned distill deletion of cross-agent memory.
