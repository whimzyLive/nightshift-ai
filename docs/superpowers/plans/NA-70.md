# On-Demand ISR for CMS Pages — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/NA-70.md](../specs/NA-70.md)
**Story:** [NA-70](https://whimzylive.atlassian.net/browse/NA-70) — Add on-demand ISR for CMS pages
**Date:** 2026-07-24
**Agents required:** web-engineer (only)

**Goal:** Add a general-purpose On-Demand ISR mechanism to `apps/marketing`: a Payload `Pages` collection pre-rendered via a generic `[slug]` App Router route, plus an `afterChange` hook that `revalidatePath`s a single edited page so it regenerates instantly on save while unchanged pages keep serving pre-rendered static HTML.

**Architecture:** A new Payload `Pages` collection (`blocks` layout builder + drafts) drives a generic `(frontend)/[slug]/page.tsx` server route. The route reads through a `lib/pages.ts` Direct-Local-API data layer (mirroring `lib/faq.ts`), pre-renders published slugs via `generateStaticParams`, and serves brand-new pages on first hit via `dynamicParams = true`. A standalone `revalidatePage` `CollectionAfterChangeHook` invalidates the current (and, on a slug change, the previous) static path.

**Tech Stack:** Next.js 16.2 (App Router, async route params), Payload CMS 3.85.2 (`@payloadcms/db-postgres`, `@payloadcms/richtext-lexical`), React 19, TypeScript, Jest + React Testing Library, Nx (`pnpm nx`), pnpm.

## Execution order

Single phase — **Phase 1 (web-engineer)** — all tasks sequential, one agent on the shared `feat/NA-70` branch. No database-administrator/platform-engineer/sync/mobile/ai-enablement phases apply: Payload owns the schema + migration (the web-engineer runs the Payload migration + typegen), and all files live under `apps/marketing/`.

## Global Constraints

- **Ownership:** every task is owned by **[web-engineer]** (workspace `apps/marketing/`). No task spans another agent.
- **Stack floors (verbatim):** `next@~16.2.10`, `payload@3.85.2`, `@payloadcms/richtext-lexical@3.85.2`, `react@^19.0.0`. Do not add dependencies.
- **UI layer (CLAUDE.md):** all UI is `.tsx` (never `.jsx`/`.js`/class components). Tailwind v4 utility classes for styling; no CSS Modules / SCSS / per-component stylesheets.
- **Static-path isolation (AC — non-negotiable):** the `[slug]` route MUST NOT read `cookies()`, `headers()`, or `searchParams`, and MUST NOT set `dynamic = 'force-dynamic'`. Any dynamic API opts the whole route out of the static cache and defeats ISR. This is the deliberate difference from the existing force-dynamic Home/Why-SDLC/FAQ routes.
- **Async route params (Next 15+):** `params` is a `Promise`; `const { slug } = await params;` before use. Reading `params.slug` synchronously yields `undefined` and 404s every page.
- **Direct Local API only:** data reads go through `getPayload({ config })` + `payload.find` in the `lib/pages.ts` layer — never an HTTP `fetch` to the CMS, never an inline `getPayload` in the route/component.
- **Graceful fallbacks:** every Payload/DB read is wrapped in `try/catch`; `getPageBySlug` returns `null`, `getPublishedPageSlugs` returns `[]` (build-time resilience, NA-16 memory). Mirror the `lib/faq.ts` convention.
- **Reserved slugs:** `home`, `faq`, `team`, `why-sdlc` — rejected at write time by the `slug` field `validate`.
- **Read filter:** both helpers filter `_status = published` (no preview path). `getPageBySlug` uses `depth: 1` (populate `Media`); `getPublishedPageSlugs` uses `depth: 0` + `select: { slug: true }`.
- **`revalidatePath` is `void`/synchronous:** never `.then`/`.catch` it or create a floating promise; wrap in plain `try/catch`, `console.error` on throw, and always `return doc`.
- **Verification commands (quality gate):** `pnpm nx lint @nightshift-ai/marketing` and `pnpm nx test @nightshift-ai/marketing` (this repo has no standalone typecheck target — `lint` + `test` are the gate).

```typescript
// apps/marketing/src/hooks/revalidatePage.ts
import { revalidatePath } from 'next/cache';

import type { CollectionAfterChangeHook } from 'payload';

/**
 * Single source of truth for the path the `[slug]` route serves a Pages doc at.
 * A doc with slug `s` renders at `/${s}` — deliberately NO `home` → `/`
 * special-case in this story (`home` is a reserved slug the collection
 * rejects; the hand-built root `/` is force-dynamic, so nothing to revalidate).
 */
export function slugToPath(slug: string): string {
  return `/${slug}`;
}

/**
 * Regenerates the edited page's static HTML on save via on-demand ISR.
 * `revalidatePath` is a synchronous, void-returning cache-invalidation call —
 * there is no promise to await. Any throw is caught + logged so a revalidation
 * failure can never fail the editor's save.
 */
export const revalidatePage: CollectionAfterChangeHook = ({ doc, previousDoc }) => {
  try {
    revalidatePath(slugToPath(doc.slug));
    if (previousDoc?.slug && previousDoc.slug !== doc.slug) {
      revalidatePath(slugToPath(previousDoc.slug));
    }
  } catch (error) {
    console.error('[revalidatePage]', error);
  }
  return doc;
};
```

```typescript
// apps/marketing/src/lib/pages.ts
import config from '@payload-config';
import { getPayload } from 'payload';

import type { Page } from '../payload-types';

export interface PageContent {
  title: Page['title'];
  content: Page['content'];
}

/**
 * The published page for `slug`, or null when none matches (route then calls
 * notFound()). depth: 1 populates each `media` block's Media upload (url/alt)
 * and any inline richText upload — depth 0 would leave a bare id and render
 * broken images. Graceful null on any Payload/DB error (lib/faq.ts convention).
 */
export async function getPageBySlug(slug: string): Promise<PageContent | null> {
  try {
    const payload = await getPayload({ config });
    const { docs } = await payload.find({
      collection: 'pages',
      where: { and: [{ slug: { equals: slug } }, { _status: { equals: 'published' } }] },
      limit: 1,
      depth: 1,
    });
    const doc = docs[0];
    if (!doc) return null;
    return { title: doc.title, content: doc.content };
  } catch (error) {
    console.error('[pages]', error);
    return null;
  }
}

/**
 * Published slugs for generateStaticParams. depth: 0 + select slug reads slugs
 * only (no relationships). Returns [] on any build-time DB failure so the build
 * still succeeds and dynamicParams covers first-hit rendering (NA-16 memory).
 */
export async function getPublishedPageSlugs(): Promise<Array<{ slug: string }>> {
  try {
    const payload = await getPayload({ config });
    const { docs } = await payload.find({
      collection: 'pages',
      where: { _status: { equals: 'published' } },
      limit: 1000,
      depth: 0,
      select: { slug: true },
    });
    return docs.map((doc) => ({ slug: doc.slug }));
  } catch (error) {
    console.error('[pages]', error);
    return [];
  }
}
```

```tsx
// apps/marketing/src/components/pages/render-blocks.tsx
import { RichText } from '@payloadcms/richtext-lexical/react';

import type { Page } from '../../payload-types';

type PageBlock = Page['content'][number];

/**
 * Server-side block registry for the CMS `[slug]` page. Maps each entry of the
 * `content` blocks array to a component by `blockType`. An unknown blockType
 * renders nothing and is logged, so a newly-added-but-unrendered block can
 * never crash the page.
 */
export function RenderBlocks({ blocks }: { blocks: Page['content'] }) {
  if (!blocks?.length) return null;
  return <>{blocks.map((block, index) => renderBlock(block, index))}</>;
}

function renderBlock(block: PageBlock, index: number) {
  switch (block.blockType) {
    case 'richText':
      return <RichText key={index} data={block.richText} disableContainer />;
    case 'media': {
      const media = block.media;
      if (!media || typeof media !== 'object') return null;
      return (
        <figure key={index} className="my-8">
          {/* eslint-disable-next-line @next/next/no-img-element */}
          <img src={media.url ?? ''} alt={media.alt ?? ''} className="w-full rounded-lg" />
          {block.caption ? (
            <figcaption className="mt-2 text-sm text-neutral-400">{block.caption}</figcaption>
          ) : null}
        </figure>
      );
    }
    default:
      console.warn('[render-blocks] unknown blockType', (block as { blockType?: string }).blockType);
      return null;
  }
}
```

> Note (spec default fill): plain `<img>` chosen over `next/image` per the spec's allowed options — the `media.url` is a Payload upload URL not registered in `next.config` image domains, and this keeps the block server-only with no client JS. Tailwind utility classes used per CLAUDE.md.

```tsx
// apps/marketing/src/app/(frontend)/[slug]/page.tsx
import { notFound } from 'next/navigation';

import { RenderBlocks } from '../../../components/pages/render-blocks';
import { getPageBySlug, getPublishedPageSlugs } from '../../../lib/pages';

import type { Metadata } from 'next';

// A brand-new CMS page (not in the build-time matrix) compiles + serves
// statically on its first visitor hit instead of 404ing.
export const dynamicParams = true;

// NOTE: deliberately NO `dynamic = 'force-dynamic'` and NO cookies()/headers()/
// searchParams reads anywhere in this route — any of those opts the route out
// of the static cache and defeats ISR (spec: static-path isolation guarantee).

export async function generateStaticParams(): Promise<Array<{ slug: string }>> {
  return getPublishedPageSlugs();
}

export async function generateMetadata({
  params,
}: {
  params: Promise<{ slug: string }>;
}): Promise<Metadata> {
  const { slug } = await params;
  const page = await getPageBySlug(slug);
  return page ? { title: page.title } : {};
}

export default async function CmsPage({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;
  const page = await getPageBySlug(slug);
  if (!page) notFound();
  return <RenderBlocks blocks={page.content} />;
}
```
