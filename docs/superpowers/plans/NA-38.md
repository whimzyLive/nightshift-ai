# Full FAQ Page — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/NA-38.md](../specs/NA-38.md)
**Story:** NA-38 — Build the full FAQ page (Epic NA-29, 5 pts)
**Date:** 2026-07-12
**Agents required:** web-engineer (single-agent story — owns `apps/marketing/`)

**Goal:** Build the standalone `/faq` page rendering all twelve CMS FAQ entries as five topic-grouped solid-card accordions, numbered continuously Q.01–Q.12, single-open across the whole page, with a bottom install CTA — all content sourced from the existing Payload `faq` collection.

**Architecture:** An `async` server component (`faq/page.tsx`) awaits a new grouped Payload Local-API reader (`getFaqPageGroups()` in `lib/faq.ts`), server-renders each Lexical `answer` via `<RichText/>`, and hands flat groups to a single client accordion that holds one open-key across all five cards. The row visual/animation contract is shared with the home accordion by extracting `FaqRow` into a reusable module so the two accordions cannot drift.

**Tech Stack:** Next.js (App Router, server components), Payload 3 Local API, `@payloadcms/richtext-lexical/react`, `@nightshift-ai/ui` (Eyebrow / InstallSnippet / CtaButton), Tailwind v4, Jest + React Testing Library.

## Global Constraints

- All UI files are `.tsx` (TypeScript + JSX) — never `.jsx`/`.js`/class components. Styling via Tailwind v4 utility classes only; no CSS Modules / SCSS / per-component stylesheets. (root CLAUDE.md)
- No `any`. Consume the generated `Faq` type from `apps/marketing/src/payload-types.ts`. (spec Data hook)
- No schema/collection/seed/CMS changes — the `faq` collection is consumed read-only. (spec Out of Scope)
- Data reads use the Payload Local API in-process (`getPayload({ config }).find(...)`) — no new REST/GraphQL route, no browser fetch. (spec API Surface)
- Use the full `answer` field, NOT `homeAnswer`. (spec Data Model)
- Graceful-empty: any Payload/DB failure returns `[]`; the page never throws / never 404s. (spec Error Handling)
- Reduced motion is satisfied by the site-wide `global.css` guard + `motion-reduce:transition-none` on rows — no extra JS branch. (spec Error Handling / AC5)
- Out of scope: FAQPage JSON-LD, OG/meta beyond `title`/`description`, `llms.txt`, search/filter/deep-link/expand-all, analytics. (spec Out of Scope)
- All tasks run on the existing `feat/NA-38` branch — one agent, serial.
- Test invocation: `pnpm nx test marketing` (run a single file with `--testPathPattern`). Tests are `*.spec.ts` / `*.spec.tsx`; mock `@payload-config` with `{ virtual: true }` and `payload`'s `getPayload` (see existing `lib/faq.spec.ts`).

- [ ] **Step 3: Implement `getFaqPageGroups`** in `apps/marketing/src/lib/faq.ts` (append below `getHomeFaqs`):

  ```typescript
  type FaqGroup = Faq['group'];

  export interface FaqPageItem {
    id: Faq['id'];
    question: Faq['question'];
    answer: Faq['answer'];
    number: number;
  }

  export interface FaqPageGroup {
    key: FaqGroup;
    eyebrow: string;
    items: FaqPageItem[];
  }

  /** Source of truth: the faq collection's `group` select option labels. */
  const FAQ_GROUP_EYEBROW: Record<NonNullable<FaqGroup>, string> = {
    positioning: '// positioning',
    'workflow-control': '// workflow & control',
    'setup-stack': '// setup & stack',
    'trust-quality': '// trust & quality',
    'cost-license': '// cost & license',
  };

  /**
   * All FAQs for the standalone /faq page, ordered by `faqOrder`, partitioned
   * into topic groups in first-appearance order. Uses the full `answer` field
   * (not `homeAnswer`). Returns [] on any Payload/DB failure — same
   * graceful-empty convention as getHomeFaqs.
   */
  export async function getFaqPageGroups(): Promise<FaqPageGroup[]> {
    try {
      const payload = await getPayload({ config });
      const { docs } = await payload.find({
        collection: 'faq',
        sort: 'faqOrder',
        limit: 100,
        depth: 0,
      });

      const groups: FaqPageGroup[] = [];
      const byKey = new Map<FaqGroup, FaqPageGroup>();

      for (const doc of docs) {
        let group = byKey.get(doc.group);
        if (!group) {
          group = {
            key: doc.group,
            eyebrow: FAQ_GROUP_EYEBROW[doc.group],
            items: [],
          };
          byKey.set(doc.group, group);
          groups.push(group);
        }
        group.items.push({
          id: doc.id,
          question: doc.question,
          answer: doc.answer,
          number: doc.faqOrder,
        });
      }

      return groups;
    } catch (error) {
      console.error('[faq]', error);
      return [];
    }
  }
  ```

- [ ] **Step 3: Implement `full-faq-accordion.tsx`:**

  ```typescript
  'use client';

  import { useState } from 'react';

  import { FaqRow, type FaqAccordionItem } from './faq-row';

  export interface FullFaqAccordionGroup {
    key: string;
    eyebrow: string;
    items: Array<FaqAccordionItem & { number: number }>;
  }

  /**
   * The five solid-card FAQ groups. A single `openId` spans ALL groups
   * (AC2): opening any row replaces the previously open row anywhere on the
   * page; `null` = all closed (first-load state). Row numbering uses
   * `item.number` (= faqOrder) so Q.NN runs continuously across group
   * boundaries (AC1). Reduced-motion is handled by FaqRow's
   * `motion-reduce:transition-none` + the global.css guard (AC5).
   */
  export function FullFaqAccordion({ groups }: { groups: FullFaqAccordionGroup[] }) {
    const [openId, setOpenId] = useState<FaqAccordionItem['id'] | null>(null);

    return (
      <div className="flex flex-col gap-12">
        {groups.map((group) => (
          <div key={group.key} className="...solid-card classes per handoff...">
            <p className="...eyebrow classes...">{group.eyebrow}</p>
            <div>
              {group.items.map((item, i) => (
                <FaqRow
                  key={item.id}
                  index={item.number}
                  item={item}
                  isOpen={openId === item.id}
                  isLast={i === group.items.length - 1}
                  onToggle={() => setOpenId((cur) => (cur === item.id ? null : item.id))}
                />
              ))}
            </div>
          </div>
        ))}
      </div>
    );
  }
  ```

  import { FullFaqAccordion, type FullFaqAccordionGroup } from './full-faq-accordion';

  export function FaqGroups({ groups }: { groups: FaqPageGroup[] }) {

        number: item.number,
        question: item.question,
        answer: <RichText data={item.answer} disableContainer />,
      })),
    }));

    return (
      <section className="...section wrapper classes per handoff...">
        <FullFaqAccordion groups={clientGroups} />
      </section>
    );
  }
