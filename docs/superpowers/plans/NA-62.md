Two sequential phases, **one agent at a time** on the shared `feat/NA-62` branch. `ai-enablement-engineer` is normally **DEPENDENCY-FREE** in the SDLC ladder, but this story imposes a **real ordering constraint**: the sweep + wrapper script MUST land and make `prettier --check "plugins/**/*.md"` exit 0 **before** the `ci.yml` line is added — the new gate invokes the script over all files, so a `ci.yml` line landing ahead of a green tree (or a missing script) fails CI immediately. So ai-enablement-engineer runs first here; platform-engineer's one-line `ci.yml` edit lands last, in the same PR. The DB→Backend→Sync→Web→Mobile phases have **zero** tasks and are omitted.

So the tree must be green (Tasks 1–3) and the script must exist (Task 4) before Task 5 adds the invocation. All in one PR — the gate and the fix are inseparable (a gate over an unswept tree is red; a swept tree with no gate silently rots again).

- **THE SWEEP HAZARD — never blanket `prettier --write "plugins/**/*.md"` / `nx format:write`.** A blind write over the whole set is exactly the operation that shatters command-block files (the NA-56 corruption): a `` ```bash ``-fenced command block re-fenced to four backticks, an `acli … --body "…"` block split, an unterminated quote. The sweep is **per-file triage** (spec Decision 2, lines 77–89). This plan pre-classifies all 43 empirically (below), but the impl **re-runs the triage script** to reconfirm before any bulk operation, and every task ends with a per-file `prettier --check` fixed-point confirmation.
- **Prettier config is location-sensitive — triage in-tree, never on a copy outside the repo.** `prettier --write` resolves `.prettierrc` (notably `proseWrap`) and `.prettierignore` **relative to the file's directory tree**. A file copied to `/tmp` or the scratchpad formats under Prettier's *defaults*, not the repo config, producing a **false no-op** (verified during this plan's prep: a copy-outside-repo triage reported 0 changes for all 43; the same triage run in-tree — write the real file, diff, `git checkout -- <file>` to restore — reported the true diffs). Any write-on-copy triage MUST use an **in-tree** scratch (write the real file on a clean tree, capture the diff, then `git checkout --` to restore) so the repo config applies.
- **`docs/superpowers/**` is prettier-ignored** (`.prettierignore`) — the spec and this plan are **exempt**; the sweep target is **only** `plugins/**/*.md`. `plugins/` is not ignored, so `prettier --check "plugins/**/*.md"` sees all 92 plugin md files.
- **`nx` cannot do an all-files-by-path check — the gate uses plain `prettier`, deliberately.** `nx format:check` is project-graph-based (scoping flags are `--all`/`--projects`/`--libs-and-apps`/affected `--base`); `plugins/**/*.md` belong to **no nx project**, so no nx form checks every plugin md file by path (spec Decision 3, lines 104). This is a reasoned, spec-documented deviation from CLAUDE.md's nx-first guideline — **note it in the script header and the PR body** so a reviewer does not flag it as an oversight.
- **RTK masks Prettier output locally — trust the raw binary, not the wrapped command.** In this dev shell, `pnpm exec prettier --check …` is rewritten by the RTK hook and printed "All files formatted correctly" even while 43 files were dirty; `./node_modules/.bin/prettier --check …` returned the true `[warn]` list + exit 1. The GitHub Actions runner has **no** RTK (spec Decision 1, lines 70), so `pnpm exec prettier`'s exit code is authoritative **in CI**. For **local** verification during the sweep, invoke `./node_modules/.bin/prettier` directly (or read the exit code, not the RTK-massaged stdout).
- **The wrapper is a THIN wrapper around the all-files `--check` — NOT a per-file write-on-copy pipeline.** The spec (Decision 1) explicitly removed the write-on-copy detector: no scratch file, no `--file-info`/`ignored:false` guard, no cleanup trap. The script's body is one `prettier --check` call plus a fail-fast empty-glob guard and a prettier-availability guard, mirroring `check-agent-skill-preloads.sh`'s shape (`set -uo pipefail`, empty-set hard-fail, offender listing, `exit 1`/`exit 0`+"OK").
- **`command-reference` auto-regen — verified N/A (Standing lesson 5).** A **script** under `plugins/sdlc/scripts/` is not a command and is not enumerated by the `command-reference` generator (input set = `plugins/*/commands/*.md` frontmatter+body → `docs/reference/commands/`, per `plugins/sdlc/refs/doc-types.md`). Adding `check-plugin-docs-format.sh` triggers **no** `command-reference` regeneration (spec lines 124–126).
- **Preload-marker guard — verified N/A.** `check-agent-skill-preloads.sh` matches its marker only in **agent** files. The new script is not an agent; the sweep edits no `plugins/sdlc/agents/*.md` file. Nothing to preserve; confirm the marker is not accidentally introduced.
- **Deferred / Out of Scope, do NOT reopen (spec "Out of Scope", lines 143–149):** non-plugin markdown (`docs/**`, `apps/`, `tools/`, `brand/`, `packages/ui/`); widening line-34's affected-only check repo-wide; non-markdown fixed-point checking; changing/pinning a different Prettier version; a pre-commit/local git hook. **`embeddedLanguageFormatting: "off"`** is permitted **only** as optional, empirically-verified defense-in-depth (spec Open Question 2) — this plan does **not** adopt it; do not add it unless it demonstrably makes the known blocks idempotent and churns no other file, and never in place of the gate.

- [ ] **Step 1: Write the wrapper script**

```bash
#!/usr/bin/env bash
# check-plugin-docs-format.sh — all-files Prettier fixed-point gate for plugin docs (NA-62).
#
# WHY THIS EXISTS (NA-62)
#   Prettier 3.6.2 is non-idempotent on certain fenced command blocks: a well-formed committed
#   plugins/**/*.md file can satisfy `format(X) != X` (state 1 — well-formed but UNSTABLE), so the
#   next unguarded `prettier --write` (a pre-commit hook, editor-on-save, or any PR touching it)
#   shatters runnable command blocks into broken four-backtick fences (the NA-56 corruption). CI's
#   line-34 format check is affected-only (`--base`-scoped), so a latent state-1 file not in the
#   current diff ships green. This gate runs the SAME native `prettier --check` predicate over ALL
#   plugins/**/*.md on every PR, closing that scope hole.
#
# WHY plain `prettier`, not `nx format:check` (deliberate CLAUDE.md nx-first deviation)
#   nx format:check is project-graph-based; plugins/**/*.md belong to no nx project, so no nx form
#   checks every plugin markdown file BY PATH. `prettier --check` over the path glob is the correct
#   (and only) tool for an all-files check of these non-project files. See NA-62 spec Decision 3.
#
# USAGE
#   bash plugins/sdlc/scripts/check-plugin-docs-format.sh
#   Exit 0 + "OK" when every plugins/**/*.md is a Prettier fixed point.
#   Exit 1 + prettier's native [warn] offender list when one or more are not (or on an empty glob /
#   missing prettier).

set -uo pipefail

fi

# Let prettier expand its own glob (quoted) so it applies .prettierignore consistently.

echo "check-plugin-docs-format: FAILED — one or more plugins/**/*.md files are not Prettier fixed points" >&2
```
