# Home hero, proof bar, and problem — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL — use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/NA-32.md](../specs/NA-32.md)
**Story:** NA-32 — Build home hero, proof bar, and problem (Epic NA-29)
**Date:** 2026-07-12
**Agents required:** `web-engineer` only.

**Goal:** Build the above-the-fold conversion path of the nightshift home page — Hero, phrase-ticker marquee, count-up Proof bar, and Problem framing — as a pixel-faithful, static-in-code recreation of the approved design handoff inside `apps/marketing`, backed by three new reusable `@nightshift-ai/ui` primitives.

**Architecture:** Reusable, page-agnostic client primitives live in `packages/ui/src/lib/nightshift/` and are re-exported from `packages/ui/src/index.ts` (mirrors `Badge`/`CtaButton`/`NightSky`). Home-specific server compositions live in the new `apps/marketing/src/components/home/` dir and assemble those primitives with the approved static copy. The page (`apps/marketing/src/app/(frontend)/page.tsx`) stays a server component; the client boundary lives only inside the three primitives.

**Tech Stack:** TypeScript + React (`.tsx` only), Next.js (App Router), Tailwind CSS v4 (`@theme` tokens in `apps/marketing/src/app/global.css`) + `var(--token)` inline styles, Nx monorepo (pnpm). Tests via Vitest through `pnpm nx run-many -t test`.

## Global Constraints

Copied verbatim from the spec — every task's requirements implicitly include these:

- **Static in code, no CMS.** All copy, install commands, proof numbers, terminal script, and problem cards are hard-coded in the React tree. No Payload collection/global, migration, or seed change (NA-29 constraint).
- **Design source is authoritative.** `docs/design/marketing-site-handoff/designs/nightshift Landing.dc.html` (sections 1–3) + `tokens/*.css` + `screenshots/{01,02,03}-home.png`. Do **not** re-author copy; source it verbatim.
- **`.tsx` only** — TypeScript + JSX, no `.jsx`/`.js`/class components (CLAUDE.md UI Layer rule).
- **Tailwind v4 utilities + `var(--token)` inline styles only.** No CSS Modules, no SCSS, no per-component stylesheets. All colors/type/spacing come from the existing `@theme` tokens in `global.css` — do **not** touch the `@theme` block.
- **Sharp edges everywhere** (radius 0) except circular dots — `--radius-*` tokens are already 0.
- **Reduced motion is a hard requirement.** Each JS-driven effect self-checks `matchMedia('(prefers-reduced-motion: reduce)')` in an effect and bails before wiring listeners — mirror `night-sky.tsx` / `cursor-glow.tsx`; do not rely on the CSS guard alone.
- **Deterministic server frame for hydration.** `Terminal` renders line 1; `CountUp` renders `0` (or final value under reduced motion) on the server — hydration must match (follow `night-sky.tsx`).
- **Consume the existing layout shell.** `(frontend)/layout.tsx` already provides NightSky/CursorGlow/NavBar/Footer/`<main>` (NA-30). Do not re-add chrome.

## Open-Question defaults (applied — implement exactly these)

The spec left three open questions with suggested defaults. This plan **adopts all three defaults** — treat them as decided, not open:

1. **Full-bleed bands vs the `max-w-[var(--container-max)]` `<main>`:** let full-bleed bands (Proof bar band, Problem band tints) break out to viewport width via a negative-margin / `w-screen` escape utility, while keeping inner content centered at `--container-max`. Do **not** widen or remove the shared `<main>` container (other stories depend on it).
2. **Keyframe location:** add the `ns-marquee` and `ns-blink` keyframes to `apps/marketing/src/app/global.css`, next to the existing `ns-drift` / `ns-twinkle` keyframes (line ~297–320), so the existing `@media (prefers-reduced-motion: reduce)` guard (line ~422) already covers them. Do **not** put them in the ui package.
3. **Visual-parity tooling:** side-by-side visual review against `screenshots/01–03-home.png` (owned by the QA phase, not this plan). No strict pixel-diff threshold; font/AA differences must not fail the gate.
