## Global Constraints

Copied verbatim from the spec's Shared conventions ÔÇö every task's requirements implicitly include these:

- **Fixed-duration easing:** `EASE_OUT = [0.22, 1, 0.36, 1]` (JS tuple) Ôçö `--ease-out: cubic-bezier(0.22,1,0.36,1)` (`global.css`). Reuse for every tween. Spring-driven items (B1/B2/B3) are exempt from `EASE_OUT` but stay transform/opacity-only.
- **Duration tokens:** `--dur-fast 120ms`, `--dur-base 200ms`, `--dur-slow 360ms`, `--dur-rise 400ms`. Any fixed-duration timing maps to one of these ÔÇö no ad-hoc ms values.
- **Reduced-motion latch:** each sub-item degrades to its **final static state** using the latch **already present in its target component** ÔÇö either the `prefersReducedMotion()` post-mount `matchMedia` function or Motion's `useReducedMotion()` (night-sky only). Never introduce a third pattern.
- **Perf envelope:** animate `transform` / `opacity` (and `filter` where already used) ONLY. GPU-composited, **CLS = 0**, zero layout-triggering properties ÔÇö no width/height/top/left/margin transitions, and **no `box-shadow` animation** (paint-bound).
- **Amplitude:** "barely noticed until removed" ÔÇö keep travel/scale/skew small and restrained.
- **No new dependency** beyond the motion.dev v12 install already in `packages/ui`. No new page, no data/API/CMS/permissions/offline change.

**Quality gate (per project-context):** `pnpm nx run-many -t test` ÔÇö plus `pnpm nx run-many -t lint` for the touched projects (`@nightshift-ai/ui`, `marketing`). Run at the end of each task group.
