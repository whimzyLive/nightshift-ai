**Goal:** Ship the `/sdlc:docs` command shell plus its first mode, `sync <STORY-KEY>`, which diff-drives deterministic regeneration of frontmatter-driven reference docs + `llms.txt` and drafts gated how-to refreshes.

**Architecture:** Five Markdown edits inside `plugins/sdlc/` (command shell, new pipeline ref, three modified refs/skill/agent) plus a plugin version bump. The command layer owns the founder-confirm gate; a two-phase `knowledge-engineer` dispatch straddles it. The pipeline contract lives once in `refs/docs-pipeline.md`, referenced by both the command and the agent.

**Tech Stack:** Claude Code plugin Markdown (commands / agents / refs / skills), `plugin.json`, prettier (Markdown formatting), YAML frontmatter.

## Global Constraints

- **Single owning agent ‚Äî `ai-enablement-engineer`.** Every task touches only `plugins/**` (command, refs, skill, agent, `plugin.json`), which is the `ai-enablement-engineer` write-scope per the project-context workspace‚Üíagent table. No other domain agent participates.
- **Base branch:** `develop` (from `.claude/project/project-context.md`). Never assume `main`.
- **Delivery (this run ‚Äî /auto Workflow A Phase 2):** commit ONLY the plan doc onto the existing `feat/NA-52` branch; no plan branch, no PR, no Jira comment. (Implementation tasks below are executed in the Phase-3 impl run, not now.)
- **"Copy the shape, do not generalize":** `docs-pipeline.md` mirrors `adr-pipeline.md`'s skeleton; `docs.md` mirrors `adr.md`. Do NOT abstract a shared ref.
- **`plugins/**` is a public hosted artifact** ‚Äî never delete for local unuse; exempt from the skills.json sync trio.
- **Skill frontmatter `description` ‚â§ 1024 chars** (Claude Code plugin limit) for any modified skill/agent frontmatter.
- **Determinism / idempotence:** the deterministic regen algorithm must yield byte-identical output on re-run with no source change (AC3/AC4/AC6).
- **Story-key regex (verbatim):** `^[A-Z][A-Z0-9]*-[0-9]+$`.

## Why only one phase (DB / backend / sync / web / mobile omitted)

This is a plugin-authoring story: it introduces no database entities, no HTTP handlers, no offline-sync rules, and no web/mobile UI ‚Äî the entire deliverable is `plugins/sdlc/` Markdown + `plugin.json`, owned solely by `ai-enablement-engineer`. Phases 1‚Äì5 (database-administrator / platform-engineer / sync-engineer / web-engineer / mobile-engineer) therefore have zero tasks and are omitted. `ai-enablement-engineer` is `DEPENDENCY-FREE`: it consumes no artifact from and produces none for any domain agent.

## Execution order
