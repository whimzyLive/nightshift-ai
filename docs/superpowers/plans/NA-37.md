# Team org-chart page — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/NA-37.md](../specs/NA-37.md)
**Date:** 2026-07-12
**Agents required:** web-engineer (only active agent applicable to this web-only story)
**Goal:** Build the standalone `/team` marketing page — hero + stats line, 4-card philosophy strip, a department org chart of static agent profile cards (dashed spine + `YOU` card + five clusters), an enforced-handbook terminal, and an install CTA — as a pixel-faithful recreation of the committed design handoff, with all roster/tier/handbook content static in code.

**Architecture:** One new static data module (`components/team/roster-data.ts`) holds the fully-typed department roster, philosophy cards, and handbook steps transcribed verbatim from `team.dc.html`. Six new server components under `components/team/` compose into `(frontend)/team/page.tsx`, a synchronous server component that replaces the current placeholder stub. No CMS, DB, API, client state, or `'use client'` directive — all hover behaviour is CSS-only, gated behind `motion-safe`. Site chrome (starfield, nav, footer, cursor glow, tokens) is supplied by the NA-30 layout shell and is NOT re-rendered here.

**Tech Stack:** TypeScript, Next.js (App Router, static server render), React (server components only), Tailwind v4 (CSS-first tokens + inline `style` with token CSS variables), Nx, pnpm, Jest + RTL. UI is `.tsx` only.

## Global Constraints

- **Web-only story.** No Data Model, API, backend, mobile, or offline-sync work — those spec sections are omitted as not applicable (no persisted entity; roster/tier/handbook content is static TypeScript in code).
- **Static render, no CMS read.** The page performs zero server data fetches. Do **not** add `export const dynamic = 'force-dynamic'` and do **not** add any `lib/*.ts` reader — this page must statically prerender at build time.
- **No `'use client'` on any team component.** No React state, no client store. All hover behaviour is pure CSS (`:hover` + `motion-safe:` / `prefers-reduced-motion` guard).
- **New directory `apps/marketing/src/components/team/`** — mirrors the `home/` and `why-sdlc/` layout. Keep `roster-data.ts` separate from `components/home/team-data.ts` (different shape — phase-tree vs department-roster — do not overload one for both).
- **Fully typed, no `any`.** Use the exact `ModelTier` / `AgentProfile` / `Department` shapes from the spec (lines 70–107).
- **Reuse `@nightshift-ai/ui` primitives unchanged:** `Eyebrow`, `InstallSnippet` (`command` prop), `CtaButton` (`variant="secondary" size="lg"` + `href` → renders an anchor). Do not rebuild these and do not add any new `packages/ui/` primitive — the tier badge is a page-local element.
- **Tier facts are authoritative** — every `tier` is verified against real `model:` frontmatter under `plugins/sdlc/agents/*.md` (opus: product-manager, solutions-architect, tech-lead, principal-engineer, qa-engineer; sonnet: all others; human: you). Badge text maps `human→HUMAN`, `opus→OPUS`, `sonnet→SONNET`.
- **Charter link derivation:** `charter ↗` href = `https://github.com/whimzyLive/nightshift-ai/blob/main/plugins/sdlc/agents/{file}` when `file` is non-null; the `YOU` card has `file: null` and renders no charter link. `file` values must match actual `plugins/sdlc/agents/*.md` filenames (12 present today).
- **Styling:** Tailwind v4 CSS-first utilities + inline `style` with design-token CSS variables; sharp edges (`border-radius: 0`, except `50%` twinkle/dot circles); no CSS Modules / SCSS. Use only existing tokens named in the spec (`--surface-card`, `--surface-terminal`, `--border-default`, `--border-accent`, `--bg-void`, `--moon-100`, `--text-muted`, `--text-dim`, `--accent`, `--terra-400`, `--indigo-400`, `--green-400`, `--cyan-400`, `--red-400`, `--link`) — do NOT introduce new tokens.
- **Copy is transcribed verbatim** from the authoritative handoff `docs/design/marketing-site-handoff/designs/team.dc.html` (`renderVals().depts` + philosophy + handbook) — all `owns` / `flow` / `artifact` / `fact` / `title` / step strings exact.
- **Full-bleed sections** (philosophy strip, handbook band, CTA band) use the established breakout pattern `relative left-1/2 right-1/2 -mx-[50vw] w-screen` with an inner ~1120px centered wrapper (see `home/final-cta.tsx`, `home/team-preview.tsx`). Hero and department roster stay within the layout container.

## Execution order

Single sequential phase on the shared `feat/NA-37` branch — one agent (web-engineer), one file group at a time. No DB, platform, sync, or mobile phases apply (all omitted as not applicable per the spec's scope classification). `ai-enablement-engineer` is not engaged — no `plugins/**` or `skills/**` change; the roster reads the agent frontmatter but does not modify it.
