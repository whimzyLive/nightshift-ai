**Goal:** Drive independent, per-plugin SemVer releases (bump + git tag + CHANGELOG) for `plugins/sdlc` and `plugins/gtm` from Conventional Commits via `nx release`, with the version source of truth in each plugin's `.claude-plugin/plugin.json`.

**Architecture:** Register each plugin as a minimal Nx project so `nx release` can see it in the graph, then point a custom `VersionActions` module (under `tools/nx-release/`) at `.claude-plugin/plugin.json` instead of `package.json`. A `release` block in `nx.json` wires the group, the conventional-commits specifier, the `{projectName}@{version}` tag pattern, and per-plugin changelogs. Baseline tags are backfilled by a maintainer from `main`.

**Tech Stack:** Nx 23.0.1 (VersionActions API), pnpm, TypeScript (`.ts` module loaded directly via Nx `loadTsFile`), Conventional Commits.
