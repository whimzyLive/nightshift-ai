# Create Marketing Website — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** [docs/superpowers/specs/NA-22.md](../specs/NA-22.md)
**Story:** [NA-22](https://whimzylive.atlassian.net/browse/NA-22) — Create Marketing Website
**Date:** 2026-07-10
**Agents required:** web-engineer (sole active agent for this story)

**Goal:** Build the nightshift marketing website in `apps/marketing` (Next.js App Router + embedded Payload CMS 3) rendering every page from Payload globals, extract the design tokens into `packages/design-system`, port the UI primitives into `packages/ui`, and theme the Payload admin.

**Architecture:** Content lives in three new Payload **Globals** (`site-settings`, `home`, `why-sdlc`) on Neon Postgres. Frontend RSCs read them through the Payload **Local API** (in-process) and pass typed props down to presentational section components. Tokens live once in `packages/design-system` (Tailwind v4 CSS-first `@theme`); `packages/ui` and the marketing app consume them. No hardcoded marketing copy in components.

**Tech Stack:** TypeScript, Next.js (App Router, RSC + ISR), Payload CMS 3, `@payloadcms/db-postgres` (Neon), `@payloadcms/richtext-lexical`, Tailwind CSS v4, `next/font`, Nx + pnpm, Jest.

## Global Constraints

- **Sole owner:** web-engineer owns every path in this plan (`apps/marketing/`, `apps/marketing-e2e/`, `packages/ui/`, `packages/design-system/`, `brand/`). No DB/backend/sync/mobile/AI-config agent is in scope.
- **UI files are `.tsx` only** — never `.jsx`, `.js`, or class components (CLAUDE.md).
- **Tailwind v4, CSS-first `@theme`** for all styling. No CSS Modules, no SCSS, no per-component stylesheets. **Sole SCSS exception:** `apps/marketing/src/app/(payload)/custom.scss` (Payload admin override).
- **No hardcoded marketing copy** in any page/section component — content arrives only via typed props sourced from Payload globals (AC2).
- **No duplicated token values** — `packages/ui` imports tokens from `packages/design-system`; defines zero raw token values (AC6).
- **Every Payload content field is optional** — Payload emits optional/`null`-able types; every section component must guard all scalar/group/array access (`content?.hero?.headline`, `items ?? []`) and skip the Lexical renderer on an empty tree.
- **Shared slug constants only** — never pass ad-hoc slug strings to the Local API (it throws on an unregistered slug).
- **Generated artifacts are never hand-edited:** `apps/marketing/src/payload-types.ts` (`payload generate:types`) and the Postgres migration (`payload migrate:create` → `payload migrate`).
- **Rich text used sparingly** — only: home `faq.items[].answer`; why-sdlc `hero.intro`, `argumentSections[].body`, `faq.items[].answer`. All other multi-line bodies are plain `textarea` → `string`.
- **Both pages use ISR:** `export const revalidate = 60`.
- **Fonts self-hosted via `next/font/google`** (Inter + JetBrains Mono) — no render-blocking Google Fonts `@import` anywhere.
- **Nx-first:** run tasks through `pnpm nx ...`. Project test command: `pnpm nx run-many -t test`.
- **Canonical token source:** `.claude/skills/nightshift-design/tokens/*.css` — vendored copies must be value-identical (parity is CI-enforced).

## Execution order

Single phase — all tasks run sequentially on the shared `feat/NA-22` branch, one web-engineer at a time. Internal task order respects dependencies: design tokens → UI primitives → Payload content types → read helpers → app styling/fonts → section components → pages → admin theme.

- [ ] **Step 5: Write the parity check `scripts/check-tokens.mjs`.** Diff CSS custom-property declarations (lines matching `^\s*--[\w-]+\s*:`) between canonical and vendored files; normalize whitespace; **exclude the canonical `fonts.css` `@import` line** (it is intentionally not vendored). Exit 1 with a diff report on drift.

```js
import { readFileSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import path from 'node:path';

const here = path.dirname(fileURLToPath(import.meta.url));
const CANON = path.resolve(here, '../../../.claude/skills/nightshift-design/tokens');
const VENDOR = path.resolve(here, '../src/tokens');
const FILES = ['colors.css', 'fonts.css', 'typography.css', 'spacing.css', 'base.css'];

const decls = (src) =>
  src
    .split('\n')
    .filter((l) => /^\s*--[\w-]+\s*:/.test(l))
    .map((l) => l.trim().replace(/\s+/g, ' '))
    .sort();

let failed = false;
for (const f of FILES) {
  const a = decls(readFileSync(path.join(CANON, f), 'utf8'));
  const b = decls(readFileSync(path.join(VENDOR, f), 'utf8'));
  const missing = a.filter((d) => !b.includes(d)); // fonts.css: canonical has same decls; @import is not a decl, excluded
  const extra = b.filter((d) => !a.includes(d));
  if (missing.length || extra.length) {
    failed = true;
    console.error(`DRIFT in ${f}:`);
    missing.forEach((d) => console.error(`  - canonical only: ${d}`));
    extra.forEach((d) => console.error(`  + vendor only:    ${d}`));
  }
}
if (failed) {
  console.error('Token parity check FAILED.');
  process.exit(1);
}
console.log('Token parity check passed.');
```

> Note: canonical `fonts.css` currently ships only the `@import` line and no `--font-*` declarations; the vendored `fonts.css` adds `--font-sans`/`--font-mono`. Because the check compares **declaration lines only** and the canonical file has none for fonts, treat `fonts.css` as declaration-exempt in the script (skip missing/extra assertions for `fonts.css`, keep it in the loop for future canonical `--font-*` additions). Keep colors/typography/spacing/base strict.

```tsx
type ButtonProps = { variant?: 'primary' | 'secondary'; href?: string; children: React.ReactNode } & React.ButtonHTMLAttributes<HTMLButtonElement>;

export function Button({ variant = 'primary', href, children, className = '', ...props }: ButtonProps) {
  const base = 'inline-flex items-center justify-center rounded-md px-5 py-2.5 text-sm font-medium transition-colors';
  const styles = variant === 'primary' ? 'bg-accent text-strong hover:bg-accent-hover' : 'border border-strong text-body hover:bg-surface';
  const cls = `${base} ${styles} ${className}`;
  if (href)
    return (
      <a href={href} className={cls}>
        {children}
      </a>
    );
  return (
    <button className={cls} {...props}>
      {children}
    </button>
  );
}
```

```tsx
import { useState } from 'react';

export function InstallSnippet({ command }: { command: string }) {
  const [copied, setCopied] = useState(false);
  const copy = async () => {
    await navigator.clipboard.writeText(command);
    setCopied(true);
    setTimeout(() => setCopied(false), 1500);
  };
  return (
    <div className="flex items-center justify-between gap-4 rounded-md bg-surface-terminal px-4 py-3 font-mono text-sm text-body">
      <code>{command}</code>
      <button type="button" onClick={copy} aria-label="Copy install command" className="text-accent hover:text-accent-hover">
        {copied ? 'Copied' : 'Copy'}
      </button>
    </div>
  );
}
```
